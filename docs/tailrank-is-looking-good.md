# TailRank 看起来不错 

> 原文：<https://web.archive.org/web/http://www.techcrunch.com:80/2006/01/17/tailrank-is-looking-good/>

 [](https://web.archive.org/web/20221203102437/http://www.tailrank.com/) 凯文·伯顿的 [TailRank](https://web.archive.org/web/20221203102437/http://www.tailrank.com/) 在对抗博客新闻创新者 [Memeorandum](https://web.archive.org/web/20221203102437/http://tech.memeorandum.com/) 方面取得了实实在在的收益，我在这里和[写了关于](https://web.archive.org/web/20221203102437/http://www.beta.techcrunch.com/2005/10/21/memeorandum-hype-and-new-feature/)[和](https://web.archive.org/web/20221203102437/http://www.beta.techcrunch.com/2005/10/12/memeorandum-is-changing-the-web/)的文章。

Memeorandum 和 TailRank 都试图近实时地聚合博客和其他边缘内容，并根据主题/事件对内容进行分组。其结果是一份“博客世界的报纸”，它向读者提供的突发新闻远远领先于传统新闻服务，甚至是谷歌新闻(缺点是围绕最终毫无根据的故事可能会出现大量对话)。

将 TailRank 的[原始界面](https://web.archive.org/web/20221203102437/http://www.beta.techcrunch.com/2005/09/21/tailrank-a-tool-for-the-long-tail/)从九月份的一篇帖子与他们现在的[进行比较](https://web.archive.org/web/20221203102437/http://www.tailrank.com/)。除了一个更有用的界面，Tailrank 还开始对博客“对话”进行分组，不同的博客作者谈论类似的话题，这种方式我迄今为止除了 Memeorandum 以外还没有在任何地方见过。

Memeorandum 仍然更擅长突发新闻——Memeorandum 上的标题可能要一天或更晚才会出现在 TailRank 上——但 TailRank 仍然比 memeor andum 年轻得多，可能会迎头赶上。像澳大利亚的 Tinfinger 和 Blogniscient 这样的新公司也在瞄准这个领域。

这两种服务之间的核心后端差异在于，Memeorandum 的创始人加布·里维拉(Gabe Rivera)亲手挑选了最初的“种子”博客，随后通过他的系统发现了这些博客的来源(保证了质量，但牺牲了广度)，而 TailRank 的大部分内容直接来自以 OPML 格式(Bloglines 和罗约等大多数 RSS 阅读器用来存储供稿的文件格式)上传他们喜爱的博客的用户。TailRank 提供的额外覆盖面可能是其在突发新闻方面落后于 Memeorandum 的原因。
