# 罗约也有博客搜索

> 原文：<https://web.archive.org/web/http://www.techcrunch.com/2006/06/02/rojo-has-blog-search-too/>

 [](https://web.archive.org/web/20220816195623/http://www.rojo.com/) 在本周写了关于来自 [Bloglines](https://web.archive.org/web/20220816195623/http://www.beta.techcrunch.com/2006/05/31/askcombloglines-launch-blog-search/) 和[gnos](https://web.archive.org/web/20220816195623/http://www.beta.techcrunch.com/2006/06/02/the-other-blog-search-engine-that-launched-yesterday/)的新博客搜索引擎之后，克里斯奥尔登(罗约 CEO)向我指出[罗约](https://web.archive.org/web/20220816195623/http://www.rojo.com/)也在本月早些时候悄悄推出了博客搜索服务。罗约过去一直专注于构建自己的 feed 阅读器，这是它第一次真正涉足博客搜索。

看起来它几乎没有被发现:宣布该产品的罗约[博客帖子](https://web.archive.org/web/20220816195623/http://blog.rojo.com/rojoblog/2006/05/new_rojo_search.html)没有任何评论或引用通告(尽管[图书馆剪辑](https://web.archive.org/web/20220816195623/http://libraryclips.blogsome.com/2006/05/19/rojo-search-and-rojo-memedigger/)捕捉到了它)。罗约博客搜索可以从罗约[主页](https://web.archive.org/web/20220816195623/http://www.rojo.com/)右上角的搜索栏进入。

罗约尝试解决博客搜索相关性的问题(我在这篇文章的最后一段[讨论了这个问题)。搜索结果根据新鲜度、潜在订阅源的总订阅量、总“mojo”投票数(一个类似 Digg 的功能—](https://web.archive.org/web/20220816195623/http://www.beta.techcrunch.com/2006/05/31/askcombloglines-launch-blog-search/)[见此处](https://web.archive.org/web/20220816195623/http://www.beta.techcrunch.com/2006/03/16/rojo-gets-relevant/)获取描述)、人们是否给这个故事加了标签(表明他们喜欢这个故事)以及帖子被阅读的总次数进行排名。这是一种相关性的算法方法，类似于 Sphere 和 Bloglines 所做的(尽管数据点非常不同)。这似乎很有效——相关性非常好。

罗约用指数的深度来权衡质量。与 Technorati 不同，罗约只索引了几百万个博客。一个博客是否在索引中，首先是基于其订阅阅读器产品的订阅者(如果有人订阅了一个博客，它会自动出现在索引中)，其次是他们通过蜘蛛程序确定的其他高质量博客。

结果与投票和标记功能一起呈现，可以单击“+”符号查看完整的博客文章。

这只是搜索的第一步。Chris 告诉我，他们将在几周内重新推出新的界面。更多关于这个的。
