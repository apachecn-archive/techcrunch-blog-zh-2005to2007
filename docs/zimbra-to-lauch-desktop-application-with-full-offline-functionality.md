# 即将推出的 Zimbra 桌面:完整的离线功能–TechCrunch

> 原文：<https://web.archive.org/web/http://www.techcrunch.com:80/2007/03/26/zimbra-to-lauch-desktop-application-with-full-offline-functionality/>

 [](https://web.archive.org/web/20220711030900/http://www.zimbra.com/) [Zimbra](https://web.archive.org/web/20220711030900/http://www.beta.techcrunch.com/2007/01/25/zimbra-6-million-paying-customers/) 将于本周晚些时候宣布一款新的离线客户端应用 [Zimbra Desktop](https://web.archive.org/web/20220711030900/http://www.zimbra.com/desktop) 。它将允许 Zimbra 用户离线时在浏览器中访问和使用 Zimbra 的电子邮件和其他办公应用程序。

今天早上，我与 Zimbra 联合创始人兼首席执行官 Satish Dharmaraj 谈论了公司的历史和新的 Zimbra 桌面产品。[在 TalkCrunch](https://web.archive.org/web/20220711030900/http://www.talkcrunch.com/2007/03/25/zimbra-goes-offline/) 听播客。

对 web 应用程序的离线访问(同样重要的是，基于 web 的数据)是一个目前备受关注的领域。Firefox 宣布 Firefox 3 将允许网站通过访问本地数据存储离线工作。像 [Scrybe](https://web.archive.org/web/20220711030900/http://www.beta.techcrunch.com/2006/10/31/scrybe-syncing-calendar-has-launched-in-beta/) 这样的新创公司正在尝试这种离线同步。Adobe(和竞争对手 T12)刚刚发布了 Apollo 平台 T14，开发者可以在浏览器之外离线运行 HTML、javascript 和 Flash 代码。

这些产品中的大多数仍在开发中(Firefox 3)或刚刚推出早期或私人测试版(Apollo，Scrybe)。Zimbra 编写了自己的代码来处理离线功能，无论用户在线还是离线，用户体验都是一样的:在浏览器中打开 Zimbra 并使用该应用程序。

Zimbra 桌面将跨平台(Windows、Mac、Linux)和跨浏览器(Firefox、IE、Safari)提供。Zimbra web 应用程序和所有用户数据都存储在客户端计算机上(数据库是 [Apache Derby](https://web.archive.org/web/20220711030900/http://db.apache.org/derby/) )。在线模式下，数据实时同步。

Zimbra Desktop 不包括浏览器中的拖放功能(例如，将附件拖到电子邮件中)，尽管该公司表示将在未来的版本中包括该功能。

包括 Zimbra Desktop 在内的所有 Zimbra 源代码都是开源的——我希望其他 web 开发人员能够仔细看看他们是如何设计的。

Zimbra recenty 宣布他们达到了 600 万付费用户。该公司总部设在旧金山，在印度有一个十人办公室。他们已经在三轮融资中筹集了 3050 万美元，并表示他们很可能不需要筹集更多资金。
