我们看到，像 [Writely](https://web.archive.org/web/20221231063952/http://techcrunch.com/?p=192) 这样做得非常好的 Ajax web 应用程序可以[让人们的眼睛](https://web.archive.org/web/20221231063952/http://www.yardley.ca/blog/index.php/archives/2005/09/01/writely-just-explained-the-webos-to-me/#comments)看到计算的未来以及 web 2.0 在未来的地位。

当您第一次观看 [Zimbra](https://web.archive.org/web/20221231063952/http://www.zimbra.com/) 演示时，您可能会有类似的体验。Zimbra 基本上是一个基于网络的 outlook/iCal/Thunderbird 应用程序，就像 Writely 是一个基于网络的 Word 版本一样。

> 在 Zimbra，我们的目标是将电子邮件、日历、联系人和其他通信技术做到最好。我们相信，通过向社区开放该技术，我们将确保我们能够最大限度地创新、扩展以及与现有消息传递系统共存的能力。

然而，Zimbra 和 Writely 对待各自市场的方式存在一些核心差异。Writely 是一个专有的托管应用程序(尽管它们以 Word 和其他格式导入和导出)。Zimbra 是一个开源项目，目前只是以演示的形式出现——如果你想运行它，你必须在自己的服务器上运行。

因此，虽然 Zimbra 不是你可以立即开始使用的东西，但你可以在这里观看托管演示[，在这里](https://web.archive.org/web/20221231063952/http://www.zimbra.com/demo/)观看 flash 演示[。源代码可从下载页面](https://web.archive.org/web/20221231063952/http://www.zimbra.com/flash_demo/flash_demo.html)[这里](https://web.archive.org/web/20221231063952/http://www.zimbra.com/downloads/)获得。

Zimbra 还集成了邮件标记。印象很深刻，也挺美的。
