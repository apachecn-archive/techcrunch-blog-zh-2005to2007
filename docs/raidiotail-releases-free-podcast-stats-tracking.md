# RadioTail 发布免费播客统计跟踪

 [](https://web.archive.org/web/20221209113336/http://radiotail.com/) 播客统计和广告公司 [RadioTail](https://web.archive.org/web/20221209113336/http://radiotail.com/) 在本周末加州安大略省[便携式媒体博览会](https://web.archive.org/web/20221209113336/http://www.portablemediaexpo.com/)之前宣布，他们现在向任何播客免费提供播客统计跟踪服务 [Ripple](https://web.archive.org/web/20221209113336/http://www.radiotail.com/ripple) 。该公司希望使用免费服务的播客们能够注册，让 RadioTail 为他们销售或提供广告。不想使用基于网络的动态广告服务功能的出版商可以无限期地免费使用这些分析。

广告和在线多媒体分析是许多人现在最关心的问题，但这个领域相对年轻，仍在形成中。虽然一些公司显然是为了赚大公司的钱，但 RadioTail 的外观、感觉和名字都是由播客人为播客制作的平民主义统计跟踪服务。

该公司由格雷格·加兰特创建，他是“风险之声”播客的幕后人。加兰特上辈子在 CNN.com 和新光风险投资公司都工作过。迄今为止，他的新公司的出版商和广告商名单已经包括商业周刊、尼康、Palm、麦格劳-希尔建筑公司和微软。有许多播客统计服务可用，但 RadioTail 在 Ripple 上做了一些好事。该服务在文件下载层面通过简单的重定向工作，因此发布者可以随意更换主机，除了 RSS 附件下载之外还可以跟踪浏览器播放，并且不向听众提出任何要求。

Ripple 的统计数据每小时更新一次，包括听众的地理位置和许多其他细节，以 RSS 格式(非常好)提供，并以 HTML(不像一些竞争对手的 Flash)显示，便于打印。stat viewing dashboard 提供了一个非常长的列表，列出了显示您的数据的各种方式，该公司正在积极努力增加数据的技术实用性，以及其直接的广告销售潜力。

不幸的是，电子表格格式的导出属于该公司打算建立的功能列表，但现在还不可用。同样，免费的 Ripple 服务并不跟踪完全下载和部分下载，尽管 RadioTail 广告投放服务的客户能够看到播放完整音频文件的听众比例。

播客分析领域的其他参与者包括面向高端大出版商的 [Podbridge](https://web.archive.org/web/20221209113336/http://podbridge.com/) 、 [Podtrac](https://web.archive.org/web/20221209113336/http://www.podtrac.com/) (类似于 RadioTail，但具有不同的功能集)、托管服务 [Libsyn](https://web.archive.org/web/20221209113336/http://libsyn.com/) 和 RSS 供应商 [FeedBurner](https://web.archive.org/web/20221209113336/http://feedburner.com/) 。感谢咨询公司[Feedia.net](https://web.archive.org/web/20221209113336/http://feedia.net/)的家伙们对这一领域关键问题的深刻见解。

本周末的便携式媒体博览会上的其他公告将包括亚当·库里的[播客](https://web.archive.org/web/20221209113336/http://podshow.com/)的新一轮资金，但[这个故事已经被打破了](https://web.archive.org/web/20221209113336/http://venturebeat.com/2006/09/28/podshow-raises-15m-more-forpodcasting-yikes/)。除了 A 轮融资的 800 万美元，Podshow 还从尚未披露的投资者那里筹集了 1500 万美元。至少可以说，有些人持怀疑态度，但这证明其他玩家正在押注一个巨大的播客经济。在线音频广告领域还有一个非常有趣的公告将于下周发布。这种不断增长的经济将会有一个尾巴，它将需要被分析，而 RadioTail 的免费统计服务可能会为许多播客出版商提供良好的服务。
