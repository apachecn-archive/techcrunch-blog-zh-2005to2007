# 为什么我不喜欢即兴重复

> 原文：<https://web.archive.org/web/http://www.techcrunch.com:80/2005/11/20/why-i-dont-like-riffs/>

 [](https://web.archive.org/web/20230301140438/http://www.riffs.com/) [Riffs](https://web.archive.org/web/20230301140438/http://www.riffs.com/) ，一个任何内容的评论网站，上周悄然上线。

它采用了混合的 wiki/社会化书签方法。任何用户都可以添加一个 URL 来开始讨论(或者只是在没有 URL 的情况下开始讨论)，Riffs 社区对这个事情进行投票，并以维基的方式进行讨论。所有页面都有 RSS，干净的界面有一些很棒的 Ajax 特性。Riffs 也有标签，包括“常用标签”，我觉得很有意思。

那么我为什么不喜欢呢？

是集中的内容。所有的内容都保存在 Riff 的服务器上，它们需要人们去网站(以某种方式)给讨论添加内容。在这种情况下，以这种方式设计他们的服务看起来绝对疯狂。

如今，评论在网上随处可见。也是在博客上找到评论的最令人兴奋的地方。试试看——搜索“无论什么”和“评论”,你会在博客世界里看到成千上万的高质量评论。

即兴重复的方法是试图插入中间，让想谈论某个主题的人进行集中讨论。他们通过为每个页面创建 RSS 源来尽可能地保持开放，我的理解是，他们可能会像 flickr 一样，创建工具来允许人们将他们在 Riffs 上创建的内容推送到他们的博客上。

在我看来，这是他们的核心使命——成为评论的 flickr。不是每个人都开玩笑说的“flickr ”,当某个东西试图成为一个很酷的新 web 2.0 应用程序时，而是字面上，人们放置他们的评论内容的地方，然后如果他们选择这样做，就可以把它拿出来放到他们的博客上。

Fred Wilson 在他的博客上写了关于这项服务的文章，并提到了类似于我在上面提到的观点。

杰夫和我的观点:博客是写内容的地方。微格式和标签将帮助人们做到这一点。我要补充的一点是:他们已经在大规模地这样做了。

弗雷德反驳道:

> 但是作为一个生活在边缘的内容创作者，我不确定我们已经准备好让微格式、标签和社交互动为我们做所有的工作。我们会实现的，我确信这一点，但我们需要一个过渡步骤，这个步骤是感觉上集中的服务，但实际上是特定于应用程序的边缘供给器。我需要一个更好的词来形容这些服务，但现在我称它们为边缘进食者。

在这一点上，我不同意弗雷德的意见。Flickr 是一个有用的“边缘馈送器”,因为它有管理照片的工具，远远超过大多数博客拥有的工具。更重要的是，他们免费为你提供图片。

点评不一样。它们易于编写，易于发布，而且管理成本很低。我们需要的是一种服务，将所有这些伟大的内容拿出来，放到一个集中的地方进行搜索和查找(以及进一步的讨论)，但它总是会将人们指向最初的帖子。

回到 Riffs，我真的很喜欢这项服务的 wiki/communit 方面。但是他们首先需要找到一种从边缘获取数据的方法。如果他们能用它做有趣的事情，那么他们就能成功。

我知道他们为什么不这么做。所有那些公开的数据…任何人都可以拿走。如果 Riffs 用我建议的方法成功了，有什么能阻止其他人做同样的事情呢？

我的回答？完全没有。但是他们目前的方法甚至没有把他们放在游戏中。我说，要找到一种方法来聚集这些数据，并以某种方式将其与讨论联系起来，就像 Memeorandum 在较小范围内所做的那样。那你就有了一家大公司。

但你最好快点。一些人已经开始行动了。
