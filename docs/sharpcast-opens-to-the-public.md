# Sharpcast 向公众开放——TechCrunch

> 原文：<https://web.archive.org/web/http://www.techcrunch.com:80/2006/05/30/sharpcast-opens-to-the-public/>

 [](https://web.archive.org/web/20201204173323/http://www.sharpcast.com/) 总部位于硅谷的 [Sharpcast](https://web.archive.org/web/20201204173323/http://www.sharpcast.com/) ，它在三月初完成了一轮 1350 万美元的[融资](https://web.archive.org/web/20201204173323/http://www.beta.techcrunch.com/2006/03/08/975/)(两轮共 1650 万美元)，现在对公众开放。官方声明是明天——周三——但是网站现在对新注册开放。

Sharpcast 很重要，但是很难一目了然。基本想法是消除在计算机和移动设备之间同步数据的麻烦。第一个产品 Sharpcast Photos 很好地展示了这项技术。在未来，我们将会看到 Sharpcast 的产品，允许自动同步联系人、日历和文档，所有这些都来自他们的本地应用程序(outlook、ical 等)。).

然而，Sharpcast Photos 允许用户将照片上传到桌面应用程序。它现在只适用于 Windows 系统的机器，Mac 版本也将很快推出。

一旦照片在桌面应用程序中，用户进行一些选项选择，然后照片被放在网络上，同步到其他计算机和移动设备。这个过程也是全方位的。上传一张照片到网上，它就会同步到桌面(和移动设备)上。用移动设备拍照，照片会同步到网络和桌面上。

我在二月份的[演示会议](https://web.archive.org/web/20201204173323/http://www.beta.techcrunch.com/2006/02/07/a-taste-of-demo-2006/)上看到了 Sharpcast 的演示，几周前又看到了首席执行官 Gibu Thomas 和营销/业务开发总监 Allen Bush 的演示。应用程序做得非常好，同步对于移动设备来说几乎是即时的。如果说有什么不同的话，那就是 Sharpcast 的人已经等了太久才推出这款产品。令人沮丧的是，我没有发现什么可以抱怨的错误。

这就是未来。问题是夏普卡斯特是否会参与其中。如果他们执行，他们会的。
