# 宁的新玩意儿

> 原文：<https://web.archive.org/web/http://www.techcrunch.com:80/2006/02/01/nings-new-stuff/>

 [](https://web.archive.org/web/20220627083624/http://www.ning.com/) 两周前我写了一篇关于[宁](https://web.archive.org/web/20220627083624/http://www.ning.com/)的[非常关键的帖子](https://web.archive.org/web/20220627083624/http://www.beta.techcrunch.com/2006/01/20/ning-rip/)，引发了很多[讨论](https://web.archive.org/web/20220627083624/http://www.dynamicobjects.com/d2r/archives/003321.html)(商业周刊[的 Rob Hof 跟进了](https://web.archive.org/web/20220627083624/http://blogs.businessweek.com/the_thread/techbeat/archives/2006/01/whats_wrong_wit.html)我的故事，然后[撤回了](https://web.archive.org/web/20220627083624/http://www.businessweek.com/the_thread/techbeat/archives/2006/01/mea_culpa_wrong.html))。

我从讨论中得到的主要收获是，宁公司的员工对我那被公认为煽动性的职位头衔(宁——注册会计师)作出了专业的回应。随后，他们联系了我和宁社区，以便更好地交流他们的产品计划。在那之前，宁对我来说是一座坚不可摧的堡垒——我就是无法与任何比我年长的人取得联系(并不是他们欠我的)。然而，上周在帕洛阿尔托的 Ning 办公室拜访了首席执行官 Gina Bianchini 之后，我现在对该公司在不久的将来的发展方向有了一个更好的想法。我会在这里公布一些细节。

首先，宁已经有了一些专门开发和推广应用的人。我看到了一些现有应用的例子，比如 cinecrap，至少可以说是“有趣的”，特别是考虑到宁的产品还处于早期阶段。

第二，宁正在努力推出新功能，包括非常简单的工具来创建独特的不同的应用程序，而不需要任何编程技能。

目前，除了克隆一个现有的应用程序，你还需要了解 PHP 或者至少 HTML。我上周看到的，还没有准备好发布，已经足够让阿宁成为一个非常有用的平台。基本上，他们将 AJAX 拖放界面与简单的 web 表单相结合来定位模块并与之交互。“模块”可以是自己创建的，或者更有趣的是，可以是第三方网络服务，比如谷歌地图或亚马逊图书元数据。如果我以前看到过这些东西，我就不会写我以前的帖子了。

还有其他很棒的功能也即将推出，比如允许出版商投放自己的广告、托管自己的应用程序等的付费托管计划。

Ning 的核心优势是无需编程技能即可构建应用程序的新工具，以及共享的客户群，这使得新的应用程序创建者可以即时访问所有注册的 Ning 客户(您只需在 Ning 注册一次，就可以使用所有应用程序)。

正如许多人在我之前的帖子的评论中所说，市场将决定宁是否成功。但是他们绝对没有死。
