# Revver:最新英国喜剧

 [](https://web.archive.org/web/20220128104158/http://www.revver.com/) 用户创建的视频分享网站 [Revver](https://web.archive.org/web/20220128104158/http://revver.com/) 已经[与一家名为 FameTV 的新英国电视台达成了有趣的合作关系](https://web.archive.org/web/20220128104158/http://blog.revver.com/?p=289)。Revver 用户将能够选择电视广播，那些被选中的将在 [FameTV](https://web.archive.org/web/20220128104158/http://www.fametv.com/) 上显示。观众将通过短信为他们最喜欢的节目投票，Revver 的收入将与视频出版商对半分成。

这是我们在以前的交易中看到的那种伙伴关系，比如 PodShow 的 Sirius broadcasts，podcast 推动的电台 KYOU 和 Tivo 的 Rocketboom。

Revver 是非常 2.0，与后滚动静止框架广告，出版商的收入分成，社会书签集成和一个 API。不幸的是，他们经历了一段艰难的时间来建立像 YouTube 那样的临界质量。除了少数高知名度的视频系列([泽弗兰克](https://web.archive.org/web/20220128104158/http://zefrank.com/)、[问忍者](https://web.archive.org/web/20220128104158/http://askaninja.com/)、孤独女孩 15)，网络上没有太多好的内容。也许是收入共享带来了更少真正的创造性和更多利润驱动的业余爱好者，以弥补 Revver 的大部分内容。我不知道。电视合作可能有帮助，也可能没有帮助。

这种合作关系在某些关键方面不同于向顶级用户付费的趋势。就质量而言，挑选一些内容创作者并向他们支付高薪，可能比从一大堆内容中浏览并向许多人支付很少的报酬更有效。 [Blogburst](https://web.archive.org/web/20220128104158/http://blogburst.com/) ( [我们的报道](https://web.archive.org/web/20220128104158/https://beta.techcrunch.com/tag/blogburst))似乎对文本内容采取了点菜的方式。

只有时间能证明哪种方法最有效。不过就目前而言，经典英国喜剧将面临至少比以往更多的挑战。
