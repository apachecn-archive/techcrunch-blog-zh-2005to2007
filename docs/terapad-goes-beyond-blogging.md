# Terapad 超越博客

> 原文：<https://web.archive.org/web/http://techcrunch.com:80/2006/09/10/terapad-goes-beyond-blogging/>

我第一次开始写博客是在 2002 年 11 月，用的是戴夫·维纳的电台 Userland。这个博客叫做[第二次思考](https://web.archive.org/web/20150806214324/http://radio.weblogs.com/0114861/categories/technology/)，它涵盖了诸如网络服务、数字版权管理、开源、初创企业、微软和谷歌等主题。奇怪的是，回顾一些帖子，似乎没有什么真正改变，除了我在 2003 年 7 月将我的个人博客转移到 Six Apart 的 Typepad，并将我的博客名称改为 [Vecosys](https://web.archive.org/web/20150806214324/http://www.vecosys.com/) 。

现在，我再次考虑更换我的个人博客提供商。过去，出现了几个博客平台，并短暂地引起了我的注意，但进一步考察后，他们只是奉承欺骗，因为他们并没有真正提供比 Typepad 更多的东西，而且搬家的麻烦不值得花时间。

我寻找的是一个“天生”采用和/或集成了一些新的 Web 2.0 特性的博客平台。例如标签云，digg，flickr，delicious，ping-o-matic，microformats 等等。可以说，使用插件和/或原始 xhtml 代码的混合，拥有一个 Web 2.0 博客在今天是完全可能的，但遗憾的是，我又一次没有时间来设置它。

一个值得注意的例外是 Marc Canter 的[人员聚合器](https://web.archive.org/web/20150806214324/http://peopleaggregator.net/login.php)，它最近实现了一系列新的 Web 2.0 特性；联合 ID、微格式、结构化博客、社交网络等。但遗憾的是，我发现 PeopleAggregator 很难使用，主要是因为 UI 设计布局有点像 Marc 著名的夏威夷衬衫；非常丰富多彩，但同样非常繁忙，非常吵杂，让人看了不舒服。

所以在我继续寻找一个新的博客平台的过程中，上周我意外地发现了 Terapad。Terapad 是一家总部位于伦敦的自筹资金的初创公司，它清楚地看到了博客市场的现状，并得出结论认为，该市场仍有一个新参与者的空间。

> “博客最近在技术上非常活跃，但在功能方面却完全停滞不前。我们利用了这一点，将主要企业网站的所有功能添加到博客等式中。”TeraPad 首席执行官 Stephan Tual 表示，

在使用了几天之后，到目前为止我喜欢 Terapad，它不仅仅是另一个博客平台。似乎 TeraPad 已经成功地将大量 web 2.0 功能集成到一个易于使用的博客平台中，同时设法将其与在线购物、网页、论坛、图片库、求职以及大量有用的第三方服务(如 Google Analytics 和 Paypal)相结合。对于更多的技术用户，TeraPad 仍然为他们提供了直接操作底层 XHTML 和 CSS 源代码的机会，以进一步定制平台。

那么，TeraPad 真的能够提供我正在寻找的那种下一代博客平台，同时说服我取代我现有的个人博客吗？我认为，Terapad 已经走了很长一段路，他们可能会严重挑战 Live Journal、WordPress.com、Spaces 和 Typepad 等公司，但我认为他们仍然需要解决以下一些问题。

首先，托管服务提供每月 18.00 美元的固定月费，比 Typepad 贵，也比 WordPress.com 之流贵很多？其次，尽管 Terapad 允许一个博客中有多个编辑，但它不允许一个账户中有多个博客。第三，从 Typepad 和 Radio Useland 导入我的旧帖子会很好，这样我可以将我的帖子和图片合并在一个地方。第四，同时支持 Feedburner 和 Google 的 AdSense 也会非常有用。最后，如果 TeraPad 抓住机会实现对微格式的支持，那就太好了。

好消息是 TeraPad 还有时间，因为他们刚刚开始公开测试，非常渴望获得用户反馈。
