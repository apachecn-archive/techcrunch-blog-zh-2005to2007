## 2.Technorati 更新“博客世界的状态”

上周，Technorati 用新的数据更新了他们的三月数据，博客圈的增长似乎还没有达到顶峰。我们(就像“我们博客圈”一样)平均每天大约有 1，000，000 个新帖子，即使是在新闻不多的日子。6 个月前还不到 20 万/天。博客的总数大约每五个月翻一番。这些信息简直令人震惊，即使我们对新的令人震惊的信息变得有些麻木。

## 4.定义 Web 2.0

[media tope](https://web.archive.org/web/20220516083854/http://doc.weblogs.com/2005/07/31#gettingPastWeb1x)的 Doc Searls 和 [Martin 为 Richard MacManus](https://web.archive.org/web/20220516083854/http://phaidon.philo.at/martin/archives/000298.html)[今年早些时候定义 Web 2.0 做出了有价值的努力](https://web.archive.org/web/20220516083854/http://www.readwriteweb.com/archives/002645.php)(Richard 也做了进一步的思考)。

由[蒂姆·布雷](https://web.archive.org/web/20220516083854/http://www.tbray.org/ongoing/When/200x/2005/08/04/Web-2.0)发起并由[蒂姆·奥雷利](https://web.archive.org/web/20220516083854/http://radar.oreilly.com/archives/2005/08/not_20.ht)反驳的另一条线索，争论 Web 2.0 是迷因还是人造迷因。我不够聪明，无法对此发表评论——我不得不在谷歌上查找 [meme](https://web.archive.org/web/20220516083854/http://www.google.com/search?q=define%3A+meme&sourceid=mozilla-search&start=0&start=0&ie=utf-8&oe=utf-8&client=firefox-a&rls=org.mozilla:en-US:official) 的意思，以便跟上对话。:-)

不过，我还是要说。我们这些整天每天都在思考 Web 2.0 的人无法用一句话来定义它。我们知道这很重要，而且它正在发生，但我们只是不能用语言表达出来。还没有。

## 5.弗雷德·威尔逊写了一部(另一部)杰作

当弗雷德·威尔逊写作时，成千上万的人急切地阅读它。他很聪明，不会滔滔不绝，他会说到点子上。他的文章“[张贴、订阅和标记](https://web.archive.org/web/20220516083854/http://avc.blogs.com/a_vc/2005/08/posting_subscri.html)”是他迄今为止最好的文章之一。

> 我喜欢把事情简单化。
> 
> 对我来说，写博客有三件事:
> 
> 发布、订阅和标记。
> 
> 这是三个必不可少的基本功能，它们是所有不同类型博客的基石。
> 
> 写博客不仅限于在 Blogger、Typepad、WordPress 或 Live Journal 上发布一小段文字。
> 
> 博客远不止这些。

弗雷德还强调，我们才刚刚开始，最好的东西还在后面。我们同意。

## 9.博客软件评论

比尔·马奇隆查看了博客软件，并对[可移动字体](www.sixapart.com)给予了适当的赞扬。我不反对这篇文章，但我要说的是。博客软件烂透了。还有巨大的改进空间。当我爸爸可以发布一个看起来很专业的可定制的博客，而对 html 或 CSS 或 RSS 或 Atom 或类别或标签或其他任何东西一无所知时，然后和我谈谈博客软件有多棒。抱歉，伙计们，现在还不是开始自我表扬的时候。有人需要像[blink sale](https://web.archive.org/web/20220516083854/http://www.blinksale.com/)([profile](https://web.archive.org/web/20220516083854/http://www.beta.techcrunch.com/?p=133))为发票所做的那样为博客做些什么。然后我们会在某个地方。用 Ajax 给我一个用户界面！

> 如果你有一份时事通讯，一个俱乐部或组织网站，或者一份针对小众读者的在线出版物，那么博客软件就是完美的。这就是我想要建立的网站所需要的:内置读者评论、RSS 聚合、基于模板的文章条目，以及无需在每个 HTML 页面中查找和替换即可进行样式更改的能力。
