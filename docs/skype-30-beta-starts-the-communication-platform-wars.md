# Skype 3.0(测试版)开启通信平台大战。

> 原文：<https://web.archive.org/web/http://techcrunch.com:80/2006/11/10/skype-30-beta-starts-the-communication-platform-wars/>

【**更新**:想看更多的秘密 skype 表情？[看到这里](https://web.archive.org/web/20130627213710/http://uk.techcrunch.com/2006/11/11/more-secret-skype-emoticons/)。

本周，Skype 3.0 ( [beta](https://web.archive.org/web/20130627213710/http://www.skype.com/download/skype/windows/skype3beta.html) )发布，表面上看，除了一个新的插件管理器之外，没有什么新东西值得兴奋。这个新版本的真正变化似乎发生在幕后，Skype 一直在开发 [Skype4Java](https://web.archive.org/web/20130627213710/http://share.skype.com/sites/devzone/) 和 [Skype4COM](https://web.archive.org/web/20130627213710/https://developer.skype.com/Docs/ApiDoc/Using_the_Skype_API_on_Windows) ，这是使用 Skype API 的新包装库。

> “Skype4Java 使开发人员能够在多个平台上创建插件，并使用良好的 ide、库和框架。有了 Skype4COM，您可以在任何 ActiveX 环境中为 Skype 编程，如 Visual Studio 和 Delphi，并使用一系列脚本语言，如 Visual Basic、JavaScript 和 PHP。

> 在 1.1 版中，Skype4COM 增加了一个安全层，使您能够在 Internet Explorer 网页中添加 Skype 功能。此功能引入了许多窗口来管理和自定义 Internet Explorer 安全设置。新的 XPCOM 包装器将把这种能力扩展到 Mozilla Firefox 浏览器，并将很快用于 beta 测试。"

我认为正是 Skype 3.0 的这些变化使得这个测试版对 Skype 未来的成功如此重要。随着这些变化，Skype 将自己定位为 IM 开发者的多操作系统通信平台，拥有强大的第三方[插件]生态系统。基本上，有了 Skype 3.0，第三方现在可以使用 Skype 协议和服务轻松地创建插件和混搭，从而创建更新的基于 P2P 的应用程序。

当然，Skype 并没有成为一个完全开放的通信平台，因为底层的 P2P 协议本身(第 5 版)仍然是专有的。虽然，一家中国初创公司声称已经通过逆向工程破解了协议，如果这是真的，可能会迫使 Skype 采取先发制人的攻击，变得更加开放，易贝(Skype 的所有者)和谷歌甚至在最近的联合新闻稿中暗示

> “从不久的将来开始，Skype 将为用户提供下载谷歌工具栏的选项，Skype 将在工具栏上添加一个自定义按钮。两家公司还将通过**开放标准**探索 Skype 和 Google Talk 之间的**互操作性**，以实现文本、聊天和在线状态

Skype 协议的第 6 版是否可能采用文本、聊天和在线状态的开放标准，如 [XMPP](https://web.archive.org/web/20130627213710/http://www.xmpp.org/) ，以实现 Skype + GTalk 的互操作性？现在，我猜想 Skype 可能会把互操作性看得比吸引开发者使用他们专有的通讯平台的直接目标更次要。

![skype_extras.jpg](img/363a57b2beae7035ab911daba04a1b2e.png)

不出所料，这种开发者平台策略并不独特，因为雅虎和微软已经围绕各自的 IM 客户端扩展了服务，并允许第三方应用程序集成其中。从许多方面来看，随着 3.0 版本的发布，我感觉 Skype 宣布正式进入“**通信平台大战**”。

作为回报，史蒂夫·鲍尔默(Steve Ballmer)本周宣布，微软打算发起**全面突击(？)**在未来一年的 VoIP 市场上。雅虎在 Web 2.0 大会上宣布，他们将把雅虎即时通讯(已经有了 VoIP 功能)直接整合到新的雅虎邮箱测试版中，从而将他们的产品整合到一个单一的界面中。

这三种通信功能(IM、VoIP + email)的发展已经不可避免，希望 SMS 集成也将成为这种新的通用通信客户端的一个特征？

迈克·阿灵顿对 TechCrunch.com 进行了全面报道，他说:

> “用户将很快习惯于根据“在线状态”在电子邮件和即时消息之间快速切换——不管他们与之交流的人是否在线。IM 对话最终将像电子邮件一样存档和存储，允许用户将旧对话拖到侧边栏的文件夹中。”

![](img/dedcc1c77d7d35932203e6d85aedfa10.png)

那么哪个传播平台【策略】会胜出呢？雅虎、微软、谷歌还是 Skype？现在说还为时过早，但我当然希望将我的即时消息、电子邮件和网络电话结合到一个客户端中，更重要的是，我只有一个联系人数据库，可以与我的手机同步。雅虎与他们今天的声明结合雅虎移动的最新进展，似乎在技术和商业(最大的用户群)方面都处于领先地位。

拥有 Gmail、GTalk 和 Google Mobile 的谷歌在技术上正在接近，但目前缺乏安装用户群。有趣的是，Outlook 2003 和 Messenger 长期以来在电子邮件中有一个**存在感知功能**，但这两个产品从未正确集成。至于微软，他们在 hotmail、live mail、messenger 和 communicator 上的战略/产品混乱。

![](img/700f28eea28c5936aea515611c6311a3.png)

就我个人而言，最近我开始越来越多地使用 Skype(用于 IM + VoIP ),因为这些有用的插件让 Skype 在很多地方都很有价值。例如，[网络](https://web.archive.org/web/20130627213710/http://www.skype.com/intl/en-gb/download/skypewebtoolbar/) & [电子邮件](https://web.archive.org/web/20130627213710/http://www.skype.com/intl/en-gb/download/skypeemailtoolbar/)工具栏，博客的[在线按钮](https://web.archive.org/web/20130627213710/http://www.skype.com/share/buttons/)以及最近的[手机](https://web.archive.org/web/20130627213710/http://www.skype.com/intl/en-gb/download/skype/mobile/) Skype 客户端，它们在我的 T-Mobile MDA-Pro 上运行得非常好。

再加上最近的“[为英国说话](https://web.archive.org/web/20130627213710/http://www.skype.com/campaigns/talkforbritain/)”未来六个月的免费座机通话激励，Skype 2.x 无疑变得非常引人注目。虽然我已经下载并安装了 3.0 版本，但我还没有使用任何附加应用程序。
