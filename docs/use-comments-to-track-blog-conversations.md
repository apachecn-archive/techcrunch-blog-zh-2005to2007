# 使用评论跟踪博客对话——TechCrunch

> 原文：<https://web.archive.org/web/http://www.techcrunch.com:80/2006/03/18/use-comments-to-track-blog-conversations/>

 [](https://web.archive.org/web/20220816013249/http://co.mments.com/) 乍一看，阿萨夫·阿金的[共同评论](https://web.archive.org/web/20220816013249/http://co.mments.com/)看起来像是[共同评论](https://web.archive.org/web/20220816013249/http://www.cocomment.com/)的竞争对手，这是一个允许你跟踪你在博客上留下的评论的工具(见[共同评论这里](https://web.archive.org/web/20220816013249/http://www.beta.techcrunch.com/tag/CoComment/))。

但实际上完全不同。Co.mments 让我想起了一种高度专业化的博客书签工具，专注于将你想要跟踪的所有帖子(包括评论讨论)保存在一个地方。你不需要在帖子上留下评论，就可以用 co . comment 跟踪它

最棒的是，你不用创建账户就可以使用它——它会跟踪事情，并根据一个 cookie 对它们进行组织。但是，如果您想创建一个帐户在不同的电脑或浏览器上使用，您可以。

要跟踪一篇文章，你只需安装一个书签工具，点击按钮就可以跟踪文章和评论。这将填充 co . comments 中的一个页面，该页面显示帖子摘要和帖子上的所有新评论。您还可以在这个[对话页面](https://web.archive.org/web/20220816013249/http://co.mments.com/conversations)上跟踪所有用户的所有帖子。

RSS 可用于您的个人页面和群组页面。

随着评论的发展，我希望看到一些新的功能加入进来。首先是一种标记文章的方式，同时为它们添加书签以便以后更容易搜索，以及通过标记与其他用户的书签分组。一般来说，还应该有更多的社交方面——例如，我想知道其他人(以及谁，多少人)是否也在跟踪一个帖子。

一个不可避免的问题是——你不能点击书签按钮，除非你在一篇文章的永久网址上。例如，仅仅呆在 Techcrunch.com 是行不通的。如果你不在一篇文章的永久 URL 页面上，一个很好的功能是一个包含在 URL 上的所有文章的下拉列表(通过查看页面的 RSS 很容易做到)。

该网站有点原始和不稳定。如果下降了，过一两天再检查。值了。感谢 [Gabe](https://web.archive.org/web/20220816013249/http://tech.memeorandum.com/) 的介绍。更多信息请见[罗伯特·斯考伯](https://web.archive.org/web/20220816013249/http://scobleizer.wordpress.com/2006/02/13/commentscom-turns-comments-into-rss/)。
