# Netvibes 承诺跨平台的部件兼容性

> 原文：<https://web.archive.org/web/http://www.techcrunch.com:80/2007/02/21/netvibes-promises-cross-platform-widget-compatibility/>

  微件平台的碎片化给开发者带来了一个问题，他们需要为各种[桌面微件平台](https://web.archive.org/web/20220328033621/http://www.beta.techcrunch.com/2007/01/29/desktop-widgets-101/) (Vista、Mac、谷歌、雅虎)以及在线平台如 Pageflakes 和 Netvibes(以及许多其他平台)开发并维护不同版本的微件。

W3C 有一个 1.0 小部件规范[的工作草案](https://web.archive.org/web/20220328033621/http://www.w3.org/TR/widgets/)，如果被采用，通过要求一些标准化，将使开发者的生活更容易。然而，对每个人来说最好的解决方案是一个小部件可以在每个平台上工作的世界，不管它最初是在哪里创建的。

今天在伦敦举行的[未来网络应用](https://web.archive.org/web/20220328033621/http://www.futureofwebapps.com/)会议上， [Netvibes](https://web.archive.org/web/20220328033621/http://www.netvibes.com/) 创始人塔里克·克里姆宣布他们即将发布的“香菜”将会做到这一点。新产品将被称为通用 Widget API，并将在[eco.netvibes.com/uwa](https://web.archive.org/web/20220328033621/http://eco.netvibes.com/uwa/)推出(该网站现已上线，有登陆页面，更多信息将于下周发布)。克里姆说，一旦推出，任何为 Netvibes 创建的小工具都可以在 [Vista](https://web.archive.org/web/20220328033621/http://msdn.microsoft.com/library/default.asp?url=/library/en-us/sidebar/sidebar/overviews/gdo.asp) 、 [Google](https://web.archive.org/web/20220328033621/http://desktop.google.com/dev/index.html) 、 [Mac](https://web.archive.org/web/20220328033621/http://developer.apple.com/macosx/dashboard.html) 和 Opera 平台上运行。对雅虎 Widgets 和其他平台的支持将会很快跟上。

一段 javascript 嵌入代码就可以将这个小部件添加到任何支持的平台上。代码将识别平台，并在小部件中为该平台运行适当的代码。一旦芫荽推出，网站将能够为大多数平台创建和推广单个插件嵌入代码。

克里姆向我展示了运行在苹果电脑和谷歌上的 Netvibes widgets。截图如下。对于熟悉 Netvibes 模块/小部件的外观和感觉的人来说，它们会立即显现出来。

Krim 还宣布，Netvibes 将在平台发布的同时开源运行时，允许任何人扩展支持的小工具平台的数量。预计较小的窗口小部件平台会加入进来。

Netvibes 现在声称有 1000 万活跃用户。克里姆说，这些用户中有 1/3 的人每天至少花一个小时在 Netvibes 上，10%的人几乎一整天都开着 Netvibes。Krim 还宣布 Netvibes 今年将支持 OpenID。
