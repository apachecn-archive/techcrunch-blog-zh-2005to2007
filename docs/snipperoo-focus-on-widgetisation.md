# Snipperoo 聚焦于 Widgetisation 

> 原文：<https://web.archive.org/web/http://techcrunch.com:80/2006/11/29/snipperoo-focus-on-widgetisation/>

上周，我在曼彻斯特参加了 NW StartUp 2.0(T1)，与来自[Snipperoo(T3)的 Ivan Pope 共同展示，我一直在跟踪他的公司。Ivan 谈到了他如何看待 widgets 改变我们与 web 交互的方式，以及他的新公司 Snipperoo 如何开发了一种新的管理 widgets 的通用工具。](https://web.archive.org/web/20160320110437/http://snipperoo.com/)

> “我们的目标是 Snipperoo 系统可以与世界上所有的网络小工具一起工作。因此，我们的目标不是使用特定种类的小部件，而是您使用我们的小部件，这是通用的，以便选择您使用的其他小部件。你可以在任何允许嵌入 javascript 的网站上使用 Snipperoo。”

widgets 现在肯定是网络焦点，今天[华尔街日报](https://web.archive.org/web/20160320110437/http://news.google.com/news/url?sa=t&ct=us/0-0&fp=45631d8ee8e7effe&ei=b3VjReaOC4KsoQKLzKDhBA&url=http%3A//online.wsj.com/article/SB116406464019128830.html%3Fmod%3Dgooglenews_wsj&cid=0)(仅限订阅)有一篇关于 widgets 越来越受欢迎的报道……”最近 [Niall Kennedy](https://web.archive.org/web/20160320110437/http://www.niallkennedy.com/) 和 [Om Malik](https://web.archive.org/web/20160320110437/http://gigaom.com/author/om-malik/) 共同制作了 [Widgets Live！此外,](https://web.archive.org/web/20160320110437/http://www.widgetslive.com/)splash cast 还对 2007 年小工具的未来做了一些非常聪明有趣的预测。有一个预测肯定让伊万很高兴。；-)

> “widget 聚合器，如 Widgetbox 或 Snipperoo 将被雅虎这样的大型媒体公司收购。或者谷歌”

需要注意的一点是，小部件的想法并不新鲜，它们只是像许多其他 Web 2.0 想法和技术一样被更新和重新营销。例如，在 Web 1.0 时期，我们有微软的桌面项目使用 RSS 的前身 CDF 频道定义框架-向小型桌面应用程序提供信息。甚至在那之前，蒂姆·伯纳斯·李爵士在 2001 年[谈到](https://web.archive.org/web/20160320110437/http://www.sciam.com/article.cfm?articleID=00048144-10D2-1C70-84A9809EC588EF21)松散耦合的可重用软件(智能代理)有助于形成[语义网](https://web.archive.org/web/20160320110437/http://www.w3.org/2001/sw/)的一部分。

今天的小工具并不真正智能，因为小工具中的大多数信息都是以纯文本或图片的形式单向传递给我们的。让窗口小部件更有用真正需要的是双向通信，比如微软的 [SSE](https://web.archive.org/web/20160320110437/http://msdn.microsoft.com/xml/rss/sse/) initiative(简单共享扩展),也就是说，我在窗口小部件中接收数据，我或另一台机器可以在它被(重新)操作之前改变它。

因此，除了 RDF、微格式、内容标签之外，小部件是语义网的另一个构件。等等。
