# WeeWorld 与美国在线结成伙伴关系？

我经常被问到英国为数不多的全球知名初创企业的名字，但我经常忘记，WeeMee 头像的创造者 [WeeWorld](https://web.archive.org/web/20220807175205/http://www.weeworld.com/) ，是一家成功的英国公司的“全球”范例。其他的是 [Bebo](https://web.archive.org/web/20220807175205/http://www.bebo.com/) 、 [DropSend](https://web.archive.org/web/20220807175205/http://www.dropsend.com/) 和 [Last.fm](https://web.archive.org/web/20220807175205/http://www.last.fm/) 。

基本上，WeeWorld ![Sam business](img/eb781c8748b828fd243dc7bb502f2845.png "Sam business")让用户创建一个卡通人物，可以作为他们在即时通讯、博客、社交网站以及移动设备上的化身。用户支付 2 美元创建一个 WeeMee，并通过手机支付额外的角色更新。右边是我自己的“生意”WeeMee。今年 5 月，WeeWorld 从 Accel Partners 和 Benchmark Capital 获得了 1550 万美元的第二轮融资，使其能够在国际上发展其数字身份业务。首席执行官西莉亚·弗朗西斯表示，这家总部位于伦敦的公司将利用这笔资金扩大其在美国的业务，支持进一步的广告和赞助交易，并实现弗朗西斯女士戏称的“WeeWorld 统治”

果然不出所料，WeeMee 本周与 AOL 建立了战略合作伙伴关系，使英国、北美和加拿大的数百万 AOL 即时通讯(AIM)用户能够通过他们自己的个性化 WeeMee 创建和交流，从而向全球统治地位又迈进了一步。你也可以在 [MSN](https://web.archive.org/web/20220807175205/http://www.wee-mee.com/culture.htm) 、 [Skype](https://web.archive.org/web/20220807175205/http://share.skype.com/sites/en/2005/12/what_is_a_weemee.html) 上使用你的 WeeMee，朋友重逢，激动不已。

据 WeeWorld Corporate [网站](https://web.archive.org/web/20220807175205/http://www.weeworld.com/about/)称，“该公司尚未盈利”，但已经有 1000 万用户创建了 WeeMees，每月有 50 万新用户加入。

> WeeWorld 首席执行官 Celia Francis 表示:“用户可以轻松创建他们独特的 WeeMee 身份，在所有数字通信渠道中使用*——从即时消息到电子邮件、博客、网络电话、图片消息和移动 IDs 并可以根据兴趣、长相、位置、激情、热情和观点搜索全球其他 wee mee。”*

大约一个月前，当这种伙伴关系的消息流传时，我和西莉亚谈过。尽管我听到了 AOL 的暗示，但我实际上希望它是一个领先的社交网络，如 Bebo、MySpace 和/或 Piczo，鉴于 AOL 正在出售其大部分欧洲公司，并严重落后于美国三大巨头，我认为这肯定更具“战略性”和更高的价值。

我们还谈到了另外两个我认为 WeeWorld 可能会发展的战略方向。首先是支持微软名为 [CardSpace](https://web.archive.org/web/20220807175205/http://cardspace.netfx3.com/) 的元身份解决方案，它将成为 Windows Vista 的一部分，也可用于 Windows XP (SP2 + .NET3)。下面是我自己 CardSpace 的截图示例，使用我自己的 WeeMee 头像创建了两个角色——我的业务和我的个人——每个角色都有不同的地址、电子邮件和电话号码。

因此，如果我遇到接受微软的“[联合身份](https://web.archive.org/web/20220807175205/http://en.wikipedia.org/wiki/Federated_identity)”CardSpace 的网站，我可以选择使用哪个卡角色来提供我的登录凭证。注:这是我创建的一个模型，目前还不是 WeeWorld 提供的服务。]当然，如果这是可能的，我相信 WeeWorld 没有理由不将这种支持扩展到其他身份解决方案，例如 [OpenID](https://web.archive.org/web/20220807175205/http://openid.net/) 、 [Yadis](https://web.archive.org/web/20220807175205/http://yadis.org/wiki/Main_Page) 或 [sxip](https://web.archive.org/web/20220807175205/http://www.sxip.com/) 。
