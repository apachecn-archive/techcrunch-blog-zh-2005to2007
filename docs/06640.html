Skype 3.0(测试版)开启通信平台大战。TechCrunch

本周，Skype 3.0 ( beta )发布，表面上看，除了一个新的插件管理器之外，没有什么新东西值得兴奋。这个新版本的真正变化似乎发生在幕后，Skype一直在开发 Skype4Java 和 Skype4COM ，这是使用Skype API的新包装库。

“Skype4Java使开发人员能够在多个平台上创建插件，并使用良好的ide、库和框架。有了Skype4COM，您可以在任何ActiveX环境中为Skype编程，如Visual Studio和Delphi，并使用一系列脚本语言，如Visual Basic、JavaScript和PHP。

在1.1版中，Skype4COM增加了一个安全层，使您能够在Internet Explorer网页中添加Skype功能。此功能引入了许多窗口来管理和自定义Internet Explorer安全设置。新的XPCOM包装器将把这种能力扩展到Mozilla Firefox浏览器，并将很快用于beta测试。"

我认为正是Skype 3.0的这些变化使得这个测试版对Skype未来的成功如此重要。随着这些变化，Skype将自己定位为IM开发者的多操作系统通信平台，拥有强大的第三方[插件]生态系统。基本上，有了Skype 3.0，第三方现在可以使用Skype协议和服务轻松地创建插件和混搭，从而创建更新的基于P2P的应用程序。

当然，Skype并没有成为一个完全开放的通信平台，因为底层的P2P协议本身(第5版)仍然是专有的。虽然，一家中国初创公司声称已经通过逆向工程破解了协议，如果这是真的，可能会迫使Skype采取先发制人的攻击，变得更加开放，易贝(Skype的所有者)和谷歌甚至在最近的联合新闻稿中暗示

“从不久的将来开始，Skype将为用户提供下载谷歌工具栏的选项，Skype将在工具栏上添加一个自定义按钮。两家公司还将通过开放标准探索Skype和Google Talk之间的互操作性，以实现文本、聊天和在线状态

Skype协议的第6版是否可能采用文本、聊天和在线状态的开放标准，如 XMPP ，以实现Skype + GTalk的互操作性？现在，我猜想Skype可能会把互操作性看得比吸引开发者使用他们专有的通讯平台的直接目标更次要。

不出所料，这种开发者平台策略并不独特，因为雅虎和微软已经围绕各自的IM客户端扩展了服务，并允许第三方应用程序集成其中。从许多方面来看，随着3.0版本的发布，我感觉Skype宣布正式进入“通信平台大战”。

作为回报，史蒂夫·鲍尔默(Steve Ballmer)本周宣布，微软打算发起全面突击(？)在未来一年的VoIP 市场上。雅虎在Web 2.0大会上宣布，他们将把雅虎即时通讯(已经有了VoIP功能)直接整合到新的雅虎邮箱测试版中，从而将他们的产品整合到一个单一的界面中。

这三种通信功能(IM、VoIP + email)的发展已经不可避免，希望SMS集成也将成为这种新的通用通信客户端的一个特征？

迈克·阿灵顿对TechCrunch.com进行了全面报道，他说:

“用户将很快习惯于根据“在线状态”在电子邮件和即时消息之间快速切换——不管他们与之交流的人是否在线。IM对话最终将像电子邮件一样存档和存储，允许用户将旧对话拖到侧边栏的文件夹中。”

那么哪个传播平台【策略】会胜出呢？雅虎、微软、谷歌还是Skype？现在说还为时过早，但我当然希望将我的即时消息、电子邮件和网络电话结合到一个客户端中，更重要的是，我只有一个联系人数据库，可以与我的手机同步。雅虎与他们今天的声明结合雅虎移动的最新进展，似乎在技术和商业(最大的用户群)方面都处于领先地位。

拥有Gmail、GTalk和Google Mobile的谷歌在技术上正在接近，但目前缺乏安装用户群。有趣的是，Outlook 2003和Messenger长期以来在电子邮件中有一个存在感知功能，但这两个产品从未正确集成。至于微软，他们在hotmail、live mail、messenger和communicator上的战略/产品混乱。

就我个人而言，最近我开始越来越多地使用Skype(用于IM + VoIP ),因为这些有用的插件让Skype在很多地方都很有价值。例如，网络 & 电子邮件工具栏，博客的在线按钮以及最近的手机 Skype客户端，它们在我的T-Mobile MDA-Pro上运行得非常好。

再加上最近的“为英国说话”未来六个月的免费座机通话激励，Skype 2.x无疑变得非常引人注目。虽然我已经下载并安装了3.0版本，但我还没有使用任何附加应用程序。
