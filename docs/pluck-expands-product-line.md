# Pluck 扩展产品线 TechCrunch

> 原文：<https://web.archive.org/web/http://www.techcrunch.com:80/2005/09/16/pluck-expands-product-line/>

 [拔毛](https://web.archive.org/web/20221002132035/http://www.pluck.com/) ( [以前的](https://web.archive.org/web/20221002132035/http://www.beta.techcrunch.com/?p=89) [简介](https://web.archive.org/web/20221002132035/http://www.beta.techcrunch.com/2005/08/11/update-pluck-new-product-releases/))正在悄悄地构建一套非常好的 web 2.0 产品(也见[阴影](https://web.archive.org/web/20221002132035/http://www.beta.techcrunch.com/?p=92))。

他们在 9 月 12 日发布了两款新产品——InSite RSS 和 InSite Blogging。这两款产品都面向在线内容出版商，尤其是在线报纸。两者都能为 Pluck 带来可观的收入，因为它们对这些服务收费。Pluck 还宣布了这些产品的第一个客户。

我通过电子邮件与 pull 的首席执行官 Dave Panos 讨论了这些产品和他们的第一笔客户交易。

## InSite RSS

InSite RSS 是一种工具，允许网站将 RSS 提要内容直接集成到其网站中，从而显著增加用户粘性。

> **InSite RSS:** 允许出版商为他们的读者提供个性化的新闻服务，利用数百万可用的 RSS 源。借助 InSite RSS，门户可以提供“我的新闻”风格的服务，让用户能够控制提要发现、订阅和消费。InSite RSS 保持网站与主要门户网站的同等地位，并确保所有提要源提供者都被适当地引用和链接。InSite RSS 利用了 pull 屡获殊荣的消费者 RSS 技术和 pull 的 FeedFinder 服务来发现 RSS 提要。

## InSite 博客

使用 InSite 博客，网站用户/读者可以创建自己的博客。对于专注于本地的网站，如在线报纸，这可能是一个非常有效的工具，可以增加粘性，以及增加关于本地问题的内容创作。虽然编辑控制可能是一个问题(正如《洛杉矶时报》痛苦地发现的那样)，但这可能是拥有在线业务的报纸停止多年来不得不遭受的客户出血的一种方式。

> **InSite Blogging:** 一个托管、管理的解决方案，使出版商能够提供社区博客作为其网站的一部分。该服务允许出版商和门户网站向读者提供功能齐全的公民新闻节目，包括照片、社区工具、标记、分类、管理和报道。一旦嵌入到网站中，该服务将打开新的内容来源和上下文广告空间，同时建立读者忠诚度和页面浏览量。InSite Blogging 建立在 2005 年初从 EasyJournal 收购的技术上，easy journal 是一个拥有数十万活跃博客作者的第三代社区。
