# MySpace 有点不耐烦了

> 原文：<https://web.archive.org/web/http://www.techcrunch.com:80/2007/01/18/myspace-is-getting-a-little-testy/>

[![](img/db1d4f69f871b6004ae921cc1defa192.png)](https://web.archive.org/web/20211026100505/http://www.myspace.com/) 今天早上，MySpace 出现了神秘的宕机。它只持续了 2.5 个小时，网站本身的表现基本正常。所有发生的事情是这样的:任何人试图在网站上添加 Flash 小部件，或者通过插入的链接显示图像，或者嵌入任何类型的代码，都做不到。现有的小部件工作正常，但是没有可以添加的。如果 MySpace 用户编辑了他们的页面，任何现有的嵌入代码都会产生错误。

到目前为止，MySpace 还没有回复我们关于为什么会发生这种情况的评论。但是，无数创业公司中的一些公司的高管们正在讨论，他们依托 MySpace，[现在是最大的互联网网站](https://web.archive.org/web/20211026100505/http://www.beta.techcrunch.com/2006/12/12/its-officialish-myspace-is-biggest-site-on-internet/)，希望获得类似 YouTube 的成功。当然，都是非正式的。一个常见的评论是“我不希望 MySpace 把我们单独挑出来。”

他们所说的是，MySpace 刚刚在即将到来的大赌注游戏“小鸡”中迈出了第一步，那里的一位高级执行官正在进行一场职业生涯定义的赌注，其他人都将首先眨眼。MySpace 仍然对收购 YouTube 耿耿于怀(他们希望如此)，有传言称，他们对 [PhotoBucket](https://web.archive.org/web/20211026100505/http://www.beta.techcrunch.com/tag/photobucket) 在获得最后一轮融资之前没有就收购一事向他们发出 ping 命令感到不满。也许最近的 2000 万美元的滑动资金太多了。所有这些公司都严重依赖(轻描淡写地)将它们的内容放到 MySpace 用户页面上。

人们已经预测到这一点有一段时间了，但是直到现在 MySpace 还没有采取任何激烈的措施来切断这些服务。今天可能是一个测试，看看他们有多容易剪掉这些小部件，看看人们会有什么反应。或者，这可能只是一个错误。

如果 MySpace 真的开始永久禁止小部件和其他嵌入式代码，他们几乎肯定会说是出于安全原因。但他们将继续推进自己的竞争服务，只允许“认证”合作伙伴进入。我敢打赌，认证过程可能会涉及费用。
