# 新的 Multiply 3.0 对比 Vox–TechCrunch

> 原文：<https://web.archive.org/web/http://www.techcrunch.com/2006/11/13/the-new-multiply-30-vs-vox/>

[![](img/df8f793736f408acf2b5ca9c2d8c7cdf.png)](https://web.archive.org/web/20211022191037/http://www.multiply.com/) 以隐私为中心的多媒体社交网络服务 [Multiply](https://web.archive.org/web/20211022191037/http://multiply.com/) 今晚发布了其服务的 3.0 版本；Multiply 的目标用户是那些对与现实生活中认识的人分享内容感兴趣，并对隐私进行严格控制的用户。社交网络中的隐私也是 SixApart 最新服务 [Vox](https://web.archive.org/web/20211022191037/http://vox.com/) 的一大卖点。Vox 在媒体分享方面也很强；这两种服务的用户可以在一个地方发布文本、照片和视频。

 [](https://web.archive.org/web/20211022191037/http://vox.com/) 现在，Vox 在经过漫长的测试期后终于推出了，Multiply 也推出了新功能和新版本，这是一个好时机来问…哪个更好？如果你正在寻找漂亮的模板，以便在年轻的时髦人群和最有可能支持早期采纳者类型用户所要求的新兴技术趋势的公司之间进行选择，那么 Vox 是正确的选择。如果你寻求逻辑和强大的隐私控制，并从陌生人的在线世界中寻求庇护，那么 Multiply 是更好的设计。我更喜欢 Multiply 处理来自你的人脉网络的媒体项目和新闻。Vox 很漂亮，Multiply 更实用。

Vox 是 SixApart 的最新产品，该公司拥有 LiveJournal、MovableType 和 Typepad。我们之前的 [Vox 报道在这里](https://web.archive.org/web/20211022191037/https://beta.techcrunch.com/tag/vox)。Multiply 是一家私人公司，最近获得了 600 万美元的投资。Multiply 已经有 2 年多的历史，号称注册用户超过 300 万。

**隐私设置和内容分发**

两家公司都认为隐私设置是关键问题。Vox 上的帖子可以对任何人公开，只对朋友、家人、朋友和家人或你自己公开。

Multiply 允许更精细的控制:您的朋友、他们的朋友和他们的朋友、您的家人、他们的家人和他们的家人、您的专业联系人、他们的专业联系人和他们的专业联系人、仅朋友、家人或专业环境中的直接联系人或仅从可从外部软件导入的联系人列表中添加的特定个人。这是一个相当多的选项列表，界面使它非常可行。如果你只想和你的家人和他们的家人分享一些东西，而不想和更多的家人分享，那么我想你不太走运。滑块而不是复选框可能是有用的。

Multiply 表示，在很多情况下，你希望与距离你的联系人圈子一两步之遥的人分享内容，但你不想为了分享而将这些人添加到你的联系人列表中。这对我来说很有意义。

换句话说，如果精细的选择性曝光对你来说很重要，那么乘法是比 Vox 更好的选择。
 **贵网新闻供稿**

Multiply 用户的首页现在是朋友和家人的最新帖子。该页面可以扩展到包括朋友的朋友，甚至进一步删除连接。该服务使用一种专有算法，包括你最近查看某人帖子的频率和其他因素，以确定非直接联系人与你的关系有多密切。

这个 My Multiply 页面比杂乱的 Vox Neighborhood 页面和过于相似但更简单的 VoxWatch 页面要干净和简单得多。正如[脸书所展示的](https://web.archive.org/web/20211022191037/http://www.beta.techcrunch.com/2006/09/08/facebook-folds-in-face-of-student-revolt/)，当你让朋友的行为在用户页面上可见时——这是你需要小心做的事情。我更喜欢 Multiply 聚合这些内容的方式，尤其是考虑到最初的海报对它将被发送给谁有如此精细的控制。

显示用户朋友的相关部分使用相同的算法来确定亲密度，并让你将鼠标悬停在任何人的图像上，以查看他们的关系，给他们发送消息或阻止他们。(左边的例子。)

**媒体处理**

我更喜欢 Multiply 与多媒体一起工作的方式，尽管有一些像 iStockPhoto 集成这样的东西来说明你的作品，这是 Vox 的优点。这两项服务都允许用户从他们的电脑或一些在线服务上传照片和视频。Vox 转码视频，需要一个恼人的额外页面加载来查看媒体文件，当我测试 Vox 的视频处理时，我看到了很大的质量损失。Multiply 在 Quicktime 中播放视频，质量没有损失，但完全超出了它发布时的原始上下文。

Multiply 的照片上传刚刚在 3.0 版本中获得了重大升级。新的 Ajax 上传程序允许你通过拖放和快速旋转图片来重新排序。用起来还是挺满足的。Vox 可以更好地将照片整合到博客文章中，但是 Multiply 中的 Ajax 重新排序功能在 Vox 中却令人沮丧地缺失了。

Multiply 支持与 LiveJournal、Blogger、TypePad、Xanga 和 Windows Live Spaces 的双向交叉发布。Vox 只支持与 LiveJournal 和 Typepad 的交叉发布，这两个网站和 Vox 一样都属于 SixApart。Multiply 提供了一个屏幕保护程序，你可以下载它来显示你指定的任何用户的最新照片——对于家庭用户来说，这听起来是一个不错的功能。

这两种服务都允许你通过手机发送电子邮件，但使用 Vox，你可以获得几个不同的电子邮件地址，并进行不同的隐私设置。Multiply 上的移动贴子太简单了，我怀疑它能有多大用处。

**权利问题**

最近，我开始把关于数据导出和身份标准的问题框定为权利问题。我拥有我的数据的权利；我希望能够轻松快速地将它从一个社交网络带到另一个社交网络。如果我想在这些不同的网络上有一个单一的登录，甚至可能有多个角色(个人的，专业的)，那么我应该能够这样做。没有人在这方面做得很好，但我预计消费者会及时提出所有需求。

Multiply 将为您刻录一张包含您所有数据的 CD，并以 50 美元的价格邮寄给您。他们说，数据导出不是他们的用户想要的东西的证据是，在该选项可用的两年多时间里，只有一个人选择了该选项。我不认为这是一个很有说服力的论点。

Multiply 还告诉我，他们对支持开放 ID 标准持谨慎态度。基于浏览器的认证或其他一些单一登录系统，因为它很容易像护照一样被淘汰。我认为除了本地登录之外，用户应该有一个基于标准的单一登录选项。该公司告诉我，他们正在观察这种或那种标准是否会受到欢迎——但这是每个人都在说的。

Multiply 提供了某种程度的角色支持，因为我可以将一些内容展示给我的家人，而将其他内容展示给专业联系人。不过，能够展示完全不同的个人资料也不错。

Multiply 允许我将联系人导出为 CSV 文件。这是一个很好的开始。Vox 不太支持这种权利问题，但它刚刚推出，SixApart 的 Anil Dash 告诉我，这一切都在工作中。当涉及到 ID 问题时，该公司的 Live Journal 属性一直是开拓者。Multiply 团队来自在线体育网站和主要媒体的世界。他们来自主流背景，针对的是主流用户。SixApart 是一家由草根创新者组成的公司，在我看来，他们更倾向于 Web 2.0 社区，他们也通过 Vox 瞄准主流用户。如果你打算在这两个推出 OpenID 或[微格式](https://web.archive.org/web/20211022191037/http://www.whymicroformats.com/pages/home)的支持上下注，Vox 是最有可能的。

**结论**

这两项服务，领先的以隐私为中心的多媒体共享社交网络，都不是我所希望的那样。然而，他们在很多事情上都做得很好。两者都在网络这个非常年轻的领域开辟了新的领域。

两家公司都可能满足不同类型的客户。在某些方面，它们还不能相提并论。例如，我喜欢 Vox 低调的广告。Multiply 是在风险投资的支持下运行的，还没有推出真正的广告策略。

在深入研究了这两个系统之后，我更加认同我最近在[采访 SixApart 的 Andrew Anker](https://web.archive.org/web/20211022191037/http://www.beta.techcrunch.com/2006/10/31/vox-says-theres-still-room-for-more-and-better-social-networking-an-interview-with-sixaparts-andrew-anker/) 时表达的观点——这个领域不仅仅是 MySpace、脸书和 LinkedIn，还有大量的空间来提供更多更好的社交网络服务。

如果你想分享文字、照片和视频，并控制哪些人可以看到你的内容，这是你应该试试的两个服务。随着 3.0 版本的推出，我认为 Multiply 已经领先了。
