# Wordpress 2.0 -好与坏

> 原文：<https://web.archive.org/web/http://www.techcrunch.com:80/2005/12/28/wordpress-20-the-good-and-the-bad-2/>

我们已经将 TechCrunch 切换到了 [WordPress 2.0](https://web.archive.org/web/20221204181128/http://wordpress.org/download/) 。不是每个人都对 WordPress 博客平台的特性集感兴趣，所以我会尽量简短。

要查看新功能的概述，请参见[无症状](https://web.archive.org/web/20221204181128/http://asymptomatic.net/2005/11/29/2135/whats-new-in-wordpress-20/)。

一个很大的变化是能够使用 Ajax 从文章页面动态创建类别。这在以前是一个多步骤的过程。由于 Technorati 和其他博客搜索引擎将类别视为标签的同义词，因此对于大多数用户来说，这是一种快速方便地为帖子添加标签的快捷方式，而无需添加额外的代码。

另一大改进是增强的“查看帖子预览”功能。这完全显示了文章在博客上的样子，所有的格式和 CSS 都将被应用。这是一个受欢迎的特性。

他们还添加了一个所见即所得的富文本编辑器。我立即禁用了它，尽管许多用户会喜欢它。

现在是坏消息。新的图像上传程序是一个火车残骸。是的，他们把它移到了删除点击的帖子页。然而，以前需要几个步骤的事情现在需要五六个步骤，因为我以上传者不支持的非常特殊的方式格式化图像。它也有默认设置，如缩略图，需要额外的点击才能得到这需要修复，否则我会真的疯了。至少，只要向我显示上传图像的 URL 字符串，就能让我找回一些丢失的功能。

图像 URL 文件夹现在也每个月重置一次，所以我也不容易找到旧的图像。他们没有考虑清楚，或者根本没有考虑清楚。

在评论和图片正确显示之前，过渡花了几天时间(是的，几天)(谢谢你， [Bryan](https://web.archive.org/web/20221204181128/http://www.avalonstar.com/) ，为此)[支持网站](https://web.archive.org/web/20221204181128/http://wordpress.org/support/)提供了一些帮助，但是问题的深度和明显的用户挫折感表明，包括我在内的许多人应该在过渡我们的整个博客之前进行测试。
