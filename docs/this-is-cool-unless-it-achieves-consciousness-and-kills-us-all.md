# 这很酷，除非它达到意识并杀死我们所有人

 [](https://web.archive.org/web/20221207212620/http://www.freebase.com/) [Freebase](https://web.archive.org/web/20221207212620/http://www.freebase.com/) 今天发布，这是一家新的创业公司，旨在组织世界信息，并使其普遍可用。如果最后一点听起来很熟悉，那是因为这实际上是谷歌的使命，但 Freebase 似乎也打算这么做。

和 Google Base 一样，Freebase 也是一个庞大的数据库。数据库的目的是集中尽可能多的数据，并允许参与者自由添加和访问数据——开发人员可以通过一组 API 从 Freebase 中提取信息，并将其添加到他们的 web 应用程序中。它还在高度结构化的数据片段之间建立关系，这对于由不同实体控制的分布式数据来说是不容易做到的。 [Tim O'Reilly](https://web.archive.org/web/20221207212620/http://radar.oreilly.com/archives/2007/03/freebase_will_p_1.html) 对这项服务及其重要性进行了深入的概述。《时代》杂志的约翰·马科夫向大众解释了这一点。

奥赖利说:

> 但希望这篇文章能让你了解 Metaweb(创建 Freebase 的公司)想要的是什么:一个类似维基百科的构建语义网的系统。但是与 W3C 的语义网方法不同，它从受控的本体开始，Metaweb 采用大众分类法，在这种方法中，人们可以在混乱无序的潜在重叠断言中添加新的类别(很像标签)。
> 
> 现在，真正强大的是，所有这些类别，这些数据类型和定义它们的字段网络，为能够从数据中提取意义的应用程序提供了新的挂钩。这就是 Metaweb 成为一种语义 web 应用的原因。
> 
> 如果 Metaweb 做到了这一点，这种自下而上的方法将在数据、新类别和思维方式之间建立新的联系。在一段时间内，这可能会是混乱和矛盾的，但正如我对约翰·马科夫说的，他正在为今晚的《纽约时报》准备 Metaweb 上的故事，他们正在为全球大脑建立新的突触。

Freebase 已经从维基百科和其他来源获取数据，个人也可以填写自己的数据。目前还不知道坏数据将如何从系统中清除。

Freebase 看起来像是 [Google Base](https://web.archive.org/web/20221207212620/http://www.beta.techcrunch.com/2005/11/15/google-base-launched-yuck/) 所没有的东西:开放且有用。我想象明天在谷歌总部会有不止一个人自慰，因为他们认为“我们本可以做到这一点。”
