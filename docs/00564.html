Web 2.0的公司，第1部分

今天， Web 2.0大会拉开了序幕，会上有许多很棒的研讨会。对我们来说，最重要的是注意力信托委员会会议(在下面发布和的帖子，当然还有)，十几家公司在一个半小时内出席了Launchpad研讨会。

我对每家公司的记录如下。他们中的许多人之前已经在这里做过简介，我们希望在我们可以安排团队采访时尽快获得其余人的完整简介(如果你想和我谈谈，我是在我的笔记本电脑上有一个巨大的TechCrunch贴纸的人)(杰夫·克拉维尔也在他的笔记本电脑上有一个TechCrunch贴纸，但我不是法国人，所以你会知道不是我:-))。

为了便于管理，我将这篇文章分成两篇。这是第一部分。第二部在这里。

罗斯·梅菲尔德谈到了第一个所见即所得的维基编辑器。然而，他说这不仅仅是维基的一个工具。它是“网络开源同步编辑器”,他的愿景是它将被用于维基之外的许多网络应用。想免费试用社交文？在socialtext上提到web2con，就可以获得一年的免费五用户wiki。

戴夫·佩尔展示了Rollyo，一个自己动手的搜索引擎。

你可以创建一个小型搜索引擎，只搜索那些你信任或认为有相关内容的网站，然后根据个人搜索进行搜索。他使用了一个非常引人注目的旅游搜索示例——只搜索fodors、travelpost和frommers。保存的搜索可以是私有的，也可以是公共的并与其他人共享。

大卫·杨谈到了Joyent，这是一个针对小团体和公司的引人注目的网络套件，包括邮件、日历、联系人、文件等。，并允许开发人员在他们的数据上混合系统。大量的标签和“智能过滤器”。开放API以允许第三方应用。在这里游览。

Rajat Paharia展示了他超酷的flash平台 BunchBall 。Rajat也很友好，在当天早些时候给了我一个个人介绍。Rajat谈到了开发人员如何同时需要基础设施和分发来发布应用程序。BunchBall提供了这两者——一个灵活的flash平台(某些应用程序需要Flash 8)以及开放的API，新的第三方应用程序会自动跨平台分发。

目前的应用包括许多游戏和照片分享。Rajar还表示，Metaliq正在开发一款多用户德州扑克游戏，即将发布。

看看这个。与传言相反，Rajat在给我演示时并没有在井字游戏中打败我。他撒谎。:-)

肯·里德谈到了他的新公司，RealTravel。它是集中式的:-(带有一些非常好的标签和搜索/查找功能的用户内容:-)。

这个想法是利用用户内容和社交网络为旅游购物者创造个性化的体验，并为旅游业营销人员提供更有效的场所。因此，有望打破在线旅游业目前所处的死亡螺旋:竞相降低价格。

Satish Dharmara对ZimbraZimbra(profile)做了一个绝对出色的介绍，尽管老实说Zimbra太酷了，充满了AJax的魅力，他可以站在那里喋喋不休，观众仍然会欢呼。

Zimbra是一款“开源企业级可扩展协作服务器，具有智能在线备份和单邮箱恢复功能。它有分级存储管理”。这是什么意思？你不能从Zimbra网站上运行它，但是你可以把它安装在你自己的服务器上。这是它应该有的前景。
