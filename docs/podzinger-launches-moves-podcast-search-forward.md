# Podzinger 发布，推进播客搜索——TechCrunch

> 原文：<https://web.archive.org/web/http://www.techcrunch.com:80/2006/01/14/podzinger-launches-moves-podcast-search-forward/>

[【Podzinger】](https://web.archive.org/web/20220112064157/http://www.podzinger.com/)，我[上个月简单地写了一下](https://web.archive.org/web/20220112064157/http://www.beta.techcrunch.com/2005/12/02/transcribing-podcasts/)和另外两个服务，几天前正式[推出](https://web.archive.org/web/20220112064157/http://tech.memeorandum.com/060114/p7#a060114p7)。Podzinger 在马萨诸塞州的剑桥工作

在我上个月的文章中，不可否认的是，当 Podzinger 仍处于深度开发阶段时，我无法让它工作。我仍然有核心功能的问题。

Podzinger 使用的语音识别技术据说也被中情局和其他政府机构用来将播客音频转换成可搜索的文本。在一些[快速搜索](https://web.archive.org/web/20220112064157/http://www.podzinger.com/results.jsp?q=steve+jobs&s=&sname=)中，它似乎会返回相关结果。结果可以按日期或相关性列出。

用户可以从搜索结果中直接收听播客，或者下载文件。用户还可以通过 RSS 订阅提要或搜索结果。

这就是我遇到问题的地方。Podzinger 的一个“杀手锏”特性是能够只听包含搜索词的片段。链接出现在结果中，点击链接可以听到片段。我看到了链接，并点击了它们，但是一个也玩不到。

这个问题在 FAQ 中解决了，它建议要么使用 Real Player，要么保持耐心。然而，我得到的是实际的错误，而不仅仅是长时间的延迟。

> **当我点击一个片段时，为什么有时从我点击播放到我听到音频播放之间会有很长的延迟？**
> 包括 Quicktime 在内的大多数媒体播放器都要求在播放开始前下载文件。使用 RealPlayer 应该不会遇到这种延迟，因为它支持 HTTP1.1 协议的范围功能，这使得几乎可以在音频文件中的任何点立即开始播放。

假设他们能正常工作，这是在播客和其他音频文件中搜索内容的好方法。一个完美的新功能将是出版商和用户能够标记文件中的附加内容，并允许其他人直接跳转到该内容。

此外，Podzinger 显然是完全转录播客…虽然我似乎无法找到完整的转录在任何地方。如果转录存在，并有一个永久的网址，我想象播客会疯狂地链接到它。

Podzinger 由广告支持，也推出了一种点击付费产品，允许播客销售广告文本。
