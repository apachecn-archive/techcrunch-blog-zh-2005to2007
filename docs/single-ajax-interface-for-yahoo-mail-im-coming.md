# 雅虎邮件和即时消息的单一 Ajax 界面即将推出

今天早上，雅虎将宣布他们将把雅虎即时通讯的 Ajax 版本直接整合到新的雅虎邮箱测试版中。与谷歌今年早些时候整合 Google Talk 和 Gmail 不同，雅虎将这两种产品整合到一个界面中。雅虎表示，新功能将在未来两个月内推出。

今天早些时候，我在旧金山的 Web 2.0 峰会上看到了该产品的测试版。当处理一封新邮件时，会出现一个弹出窗口来选择地址簿建议，如果此人当前在线，还会有发送即时消息的选项。如果用户选择即时消息而不是电子邮件，则会出现一个略有不同的界面，允许两个用户互相发送即时消息。如果一个用户退出对话，另一个用户可以选择将对话自动粘贴到电子邮件中并结束对话。请参见下面的屏幕截图，了解产品的外观。这些都是在浏览器中用 Ajax 完成的。

用户将很快习惯于根据“在线状态”在电子邮件和即时消息之间快速切换——不管他们与之交流的人是否在线。IM 对话最终将像电子邮件一样存档和存储，允许用户将旧对话拖到侧边栏的文件夹中。

侧边栏中还添加了一个注释，让电子邮件用户知道他们存储的联系人中哪些人当前在线。多个当前 IM 对话由水平选项卡组织。

雅虎邮箱在全球和美国的使用量持续飙升。雅虎声称，雅虎邮箱在 2006 年前九个月获得的美国新用户比迄今为止美国 Gmail 用户总数还要多。康姆斯克支持这种说法，报告说雅虎邮箱的全球用户从 2.5 亿增加到了谷歌的 5100 万(2006 年 9 月)。

与基于网络的电子邮件用户相比，全球即时消息用户的数量仍然很少——8000 万即时消息用户对近 5 亿网络邮件用户。雅虎希望向从未尝试过 IM 的大部分雅虎网络邮件用户介绍 IM。

与 Gmail 相比，我们仍然更喜欢 Yahoo Mail，因为 Yahoo Mail 允许 POP 访问第三方电子邮件服务，而 Gmail 只允许访问 Google 自己的电子邮件服务。
