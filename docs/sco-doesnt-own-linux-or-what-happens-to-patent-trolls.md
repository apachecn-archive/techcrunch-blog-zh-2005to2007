# SCO 不拥有 Linux 或者“专利巨魔会怎么样”

自 2003 年以来，上海合作组织一直在与越来越大的敌人作战，希望加强其 UNIX 专利。好吧，法官刚刚裁定 [SCO](https://web.archive.org/web/20201128183337/https://crunchbase.com/organization/sco) 没有 UNIX 专利，Novell 拥有所有这些专利。你在这里看到的图表是当消息传来时他们的股票价格的变化。

SCO 不生产也不销售任何利益相关的产品。到目前为止，他们的作案手法很简单——在 UNIX 和 Linux 上实施可疑的专利，并把赌注押在肯定会随之而来的大笔钱上。然而，随着这一决定，他们的日子屈指可数了。

> 该公司在一份声明中表示，“尽管地区法官在重要问题上做出了有利于 Novell 的裁决，但该案件尚未经过法律系统的全面审查，我们将继续探索我们如何从这里向前发展的选项。”

伙计们，想怎么压就怎么压。你的商业模式正在消亡。

更新——我是个大笨蛋。这主要是关于版权的。“专利猴”科里·索利斯写道:

> 这实际上是一个版权问题，在 2003 年的一次诉讼中，SCO 声称 IBM 错误地复制了 SCO 的版权，从 Unix 创建了 Linux，SCO 声称 SCO 在 1995 年从 Novell 购买了 Unix 的代码。鉴于这一裁决，Novell 可以迫使 SCO 终止此案，尽管 IBM 可能对 SCO 在那家失败的合资公司负有其他义务。
> 
> 总而言之，对自由/开源软件的人来说是好消息，对上海合作组织来说是坏消息，正如股价所显示的。
