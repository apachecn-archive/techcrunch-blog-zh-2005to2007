# Sproose:人力搜索遇上 Digg

[![sproose.png](img/755fb9be1a8e0529d1417576b13765ec.png)](https://web.archive.org/web/20221006071550/http://www.sproose.com/) 问你有多少种方法可以做一个搜索引擎就像问你有多少种方法可以炒一个鸡蛋，有很多不同的方法，尽管这不是火箭科学，但你仍然可以得到一些不可食用的东西。

人力搜索是目前最时髦的~~鸡蛋争夺~~搜索引擎食谱。像 [ChaCha](https://web.archive.org/web/20221006071550/http://www.beta.techcrunch.com/tag/chacha) 这样的服务让合同工实时回答搜索查询。Jason Calacanis 车辆 [Mahalo](https://web.archive.org/web/20221006071550/http://www.beta.techcrunch.com/tag/mahalo) 本周在 alpha 发布，采用维基百科与谷歌相遇的模式，旨在为 10，000 个搜索查询提供预先编写的结果。

总部位于加州丹维尔的[sprouse](https://web.archive.org/web/20221006071550/http://www.sproose.com/)将人力搜索与 Digg 联姻。

Sproose 是一个个性化的搜索引擎，结合了社交网络和同行审核的排名，使用户能够优先考虑、定制和微调搜索，以产生相关的网络搜索结果。

Sproose 用户可以有效地对相关网站进行分类和索引，并为个人或团体定制这些网站。通过集体审核和评分，用户可以对现有网站进行分类，只收集最合适的结果。

结果还不错。目前还不清楚搜索结果最初来自哪里(我猜是谷歌)，链接优先级的社交投票功能创造了一种不同的搜索体验。视频结果来自 Blinkx 和 Sproose，索引超过 25，000 个新闻来源。能否成功是另一回事；每个人都想成为下一个谷歌，而且不乏竞争对手。老实说，我见过比斯普鲁斯更糟的。
