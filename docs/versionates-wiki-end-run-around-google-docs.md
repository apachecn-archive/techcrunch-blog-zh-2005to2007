# Versionate 的维基端围绕谷歌文档运行

 [](https://web.archive.org/web/20210928090833/http://www.crunchbase.com/company/Versionate) [ Y Combinator 的](https://web.archive.org/web/20210928090833/http://www.crunchbase.com/company/ycombinator)夏季第二支公司 [Versionate](https://web.archive.org/web/20210928090833/http://www.crunchbase.com/company/Versionate) 今日上线。它是一个文档驱动的 wiki 产品——上传几乎任何类型的文件(办公文档、图像等), Versionate 将创建一个可编辑的、类似 wiki 的页面，并提供版本和访问控制。

Versionate 是另一家 Y Combinator 公司 [Scribd](https://web.archive.org/web/20210928090833/http://www.crunchbase.com/company/scribd) 的远亲，它允许用户上传和显示 Flash 中的文档。不过，Versionate 是维基初创公司和谷歌文档的竞争对手。在在线编辑文档方面，它也比竞争对手更灵活，因为几乎任何类型的文件都可以上传(文件也可以选择文件格式导出)。

然而，该公司的目标显然是更多的企业人群，而不是谷歌文档和维基百科。该服务最有趣的部分之一是他们说他们只考虑未来的事情——在私有服务器上安装 Versionate:

> **我们可以在内部服务器上安装 Versionate 吗？**
> 
> 目前，我们只是一个托管解决方案。这有利于我们管理所有的安全细节，并让我们在一致的基础上推出更新。如果您仍然希望许可我们的软件在您自己的内部服务器上使用，请通过电子邮件直接联系我们 support@versionate.com。

对于考虑使用这些服务的公司来说，数据安全是最大的障碍。允许他们在自己的服务器上托管数据会让他们在使用时更加放心，随着时间的推移，这可能会对 Microsoft Exchange Server/Sharepoint 构成真正的威胁。
