# 我对阅读清单的看法

> 原文：<https://web.archive.org/web/http://www.techcrunch.com:80/2005/10/16/my-thoughts-on-reading-lists/>

戴夫·维纳一直在思考一个新想法:阅读清单，最近他还写了一篇文章。

[OPML](https://web.archive.org/web/20220119174047/http://www.opml.org/) 是一个非常有用的文件结构，几乎每个使用像 bloglines 这样的提要聚合器的人都已经在使用它，而不一定知道它。大多数阅读器以 OPML 格式保存用户订阅的提要，以便于导入和导出。如果你[导出](https://web.archive.org/web/20220119174047/http://www.bloglines.com/help/faq#export)你的 OPML feed，你会得到一个你的 feed 的 XML 文件，其他 feed 阅读器可以理解。

来自阅读器的 opml 文件的问题是它们是静态的，这意味着我可以给你我的 OPML 文件，但你永远不会知道我是否添加或删除提要，除非我告诉你并给你新的文件。从我与你分享我的文件的那一刻起，你得到的只是我的提要的快照。

Dave 认为这些文件应该是动态的，这意味着我可以共享我的 opml 文件，或者他称之为我的*阅读列表*，任何订阅它的人都将始终拥有最新版本，无论我多久修改一次列表。

实现这一点只需要很少的技术——各种提要阅读器只需要同意支持动态列表，并允许人们永久共享它们。戴夫正在努力实现这一目标。

如果他成功了，我们就可以从我们信任的人那里订阅特定主题的阅读列表，找到好的订阅源也就容易多了。

弗雷德·奥利维拉[最近也写了关于这个的文章](https://web.archive.org/web/20220119174047/http://webreakstuff.com/blog/2005/10/reading-lists-a-new-way-for-opml/)。在[的一篇评论中，林育泽](https://web.archive.org/web/20220119174047/http://gaijin.wordpress.com/)写道:

> 我可以很容易地看到，这不仅是与我认识的人分享我的阅读清单的一种方式，也是与我不认识的有共同兴趣的人匹配的一种方式。如果系统能把我和其他和我有相似的科技、音乐或生活方式的人匹配起来会怎么样。这将是一种建立新联系以及加强现有联系的奇妙方式，我可以看到围绕重叠提要形成的社区。这些社区可能比围绕单个网站形成的社区更强大，因为它们有更多的共同点。

是啊，没错。一旦人们开始思考这个想法，许多其他有趣的想法就会涌现出来。

我们需要它的一个真实例子是我们的[web 2.0 工作组](https://web.archive.org/web/20220119174047/http://www.web20workgroup.com/)。每个博客的供稿都是链接的，我们有一个静态的 OPML 文件，随着新供稿的加入，我们会更新这个文件。但是，由于该文件是静态的，任何下载该文件的人都必须在添加新的提要时经常查看。让我们自动化这个过程。
