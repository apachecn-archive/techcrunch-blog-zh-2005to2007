# 使用红色 Swoosh 免费提供文件| TechCrunch

> 原文：<https://web.archive.org/web/http://www.techcrunch.com:80/2006/07/05/use-red-swoosh-to-serve-files-for-free/>

总部位于硅谷的[](https://web.archive.org/web/20230308120251/http://www.redswoosh.net/index2.php)[Red Swoosh](https://web.archive.org/web/20230308120251/http://www.redswoosh.net/index2.php)今天发布了其文件服务技术的免费广告支持版本(这句话中的链接指向新网站——redswoosh.net 将在一夜之间重定向)。

在今天之前，Red Swoosh 只对付费用户开放，他们使用类似 bittorent 的技术，通过点对点技术快速传输文件。结果是用户的文件下载速度大大加快，文件分发服务器的带宽压力几乎为零。

新产品免费提供红色 Swoosh。在下载开始之前，每第五个下载请求显示一个插播广告。顾客可以通过付费服务绕过广告。

使用红色 Swoosh 极其简单。作为内容提供商，您不需要注册帐户。你所要做的就是在任何 URL 的前面加上“http://edn.redswoosh.net ”,然后把那个链接放到你的网站上。如果您选择创建一个帐户，Red Swoosh 将向您提供下载统计报告，并在未来分享广告收入。这最后一点很重要。

红色 Swoosh 有一些限制。用户必须安装一个客户端来方便下载。这仅需要一次，任何红色 Swoosh 客户的后续下载都将有效。不过，Mac 平台还没有可用的客户端。Red Swoosh 承诺将在今年晚些时候推出。在此之前，Mac 用户仍然可以不使用 P2P 技术直接从服务器上下载文件。

Red Swoosh 成立于 2001 年，在危机期间经历了一些困难时期。他们去年对公司进行了资本重组，并从马克·库班筹集了 170 万美元。他们目前正在进行另一轮融资，首席执行官特拉维斯·卡兰尼克告诉我，融资额将在 500 万至 800 万美元之间。

另一方面，Red Swoosh 在今年早些时候正在更换办公室，并抓住机会暂时“离岸”——他们将六人公司搬到泰国甲米(Krabi)一个月，同时开发新产品(马克·库班为此开了绿灯)。在这里[和](https://web.archive.org/web/20230308120251/http://blog.redswoosh.net/2006/04/offshore-yourself.html)[这里](https://web.archive.org/web/20230308120251/http://blog.redswoosh.net/2006/05/offshoring-yourself-ground-rules.html)查看 Travis 关于该项目的博客帖子。Travis 告诉我，这是一次很棒的经历，团队比回到硅谷时更有效率。
