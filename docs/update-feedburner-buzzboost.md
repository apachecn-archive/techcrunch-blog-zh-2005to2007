**有什么新消息？**

最近, [FeedBurner](https://web.archive.org/web/20201205122105/https://crunchbase.com/organization/feedburner) 推出了一大堆[新功能](https://web.archive.org/web/20201205122105/http://www.beta.techcrunch.com/?p=15)，而 [BuzzBoost，在 7 月 11 日](https://web.archive.org/web/20201205122105/http://www.burningdoor.com/feedburner/archives/001311.html)发布，值得注意。

用他们自己的话说，*“进入 BuzzBoost。这是 FeedBurner 出版商最新的宣传和认知产品，而且是免费的。BuzzBoost 的工作是在您可以复制和粘贴 HTML 代码的任何地方重新显示您的提要内容—在 Blogger 或 TypePad 页面模板中，在公司网站上，甚至在留言板的“签名”块中。BuzzBoost 代码只是一小段 JavaScript 代码，它根据您提供的设置显示来自 FeedBurner 提要的内容项和信息。您可以控制 BuzzBoost 如何显示以下元素:*

*提要标题
*项目发布日期
*单个项目标题、标题和纯文本，或标题和原始 HTML
*纯文本项目摘录长度
*下载播客附件的链接(如果有)

我们知道，熟悉级联样式表(CSS)的出版商也希望精确控制 BuzzBoost 生成的任何内容的样式。因此，BuzzBoost 在生成的 HTML 中有正确的挂钩，以确保您可以对 BuzzBoost 的每个元素进行样式化，我们已经对此进行了记录(参见本文底部的链接。)"
[链接](https://web.archive.org/web/20201205122105/http://www.burningdoor.com/feedburner/archives/001311.html)

几天前，Qumana 的 Tris Hussey 发布了这个帖子，并立即看到了它的力量——你可以交叉推广博客，并以新的方式使用任何 RSS 源。例如，Tris 正在考虑获取潜在搜索查询的 RSS 源，并通过 buzzboost 进行推广——非常有创意！这不是新技术，它只是一个非常简单和酷的实现。

Feedburner 将根据您的参数为您创建一个 html 脚本，或者您可以获取源 html 并完全自定义它。

要使用它，如果你已经在 Feedburner 上刻录了提要，在控制面板区域点击“公开”，然后选择“buzzboost”。

![](img/058bf4e284a176d0c3640c8ba443b8a2.png)
