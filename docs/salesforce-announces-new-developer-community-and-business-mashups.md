# 独家:Salesforce Business Mashups，新的开发人员社区

![appexchange logo](img/668139ae25fb734f19172a65d7afe6aa.png)

我有幸独家看到了明天 [Salesforce](https://web.archive.org/web/20221003073320/http://www.salesforce.com/) 将在 [eTech](https://web.archive.org/web/20221003073320/http://conferences.oreillynet.com/etech/) 大会上发布的新开发者社区和工具包。多年来，Salesforce 一直站在我们现在称之为 Web 2.0 的商业领域的前沿。早在我们今天使用术语来描述这些技术和想法之前，他们就试图将思维转向软件即服务和网络即平台。Salesforce 已经成长为不仅仅是一个 CRM 应用程序，而且现在是一个在 web 上进行业务协作和交流的平台，允许开发人员在他们的平台上构建应用程序，并让他们的 399，000 多个客户群访问这些应用程序。

Salesforce 作为一个平台发展的最佳迹象是，现在超过 40%的对 Salesforce web 服务器的请求是对其 API 的 SOAP 请求。 [AppExchange](https://web.archive.org/web/20221003073320/http://www.appexchange.com/) 是 Salesforce 的应用程序平台，目前其应用程序库中列出了超过 160 个应用程序，所有这些应用程序都可供 Salesforce 客户使用。在 eTech，Salesforce 计划宣布推出他们的新开发人员社区—[AppExchange 开发人员网络](https://web.archive.org/web/20221003073320/http://developer.appexchange.com/),为开发人员提供社区、工具和资源，让他们为 app exchange 构建应用程序。Salesforce 还将宣布针对 [PHP](https://web.archive.org/web/20221003073320/http://www.salesforce.com/developer/projects_toolkits.jsp#php) (支持 PHP5 中的原生 SOAP 库——与 [Zend](https://web.archive.org/web/20221003073320/http://www.zend.org/) 合作开发)和令人兴奋的 [RubyOnRails](https://web.archive.org/web/20221003073320/http://www.rubyonrails.org/) 工具包的可用性，该工具包名为 [ActiveSalesforce](https://web.archive.org/web/20221003073320/http://activesfdc.rubyforge.org/) 。

Salesforce 将消费者领域的 Web 2.0 称为业务领域的“业务 Web”。Web 2.0 背后的技术和理念，如 Ajax、SOAP、软件即服务等。所有这些都始于商业领域，它们向消费者领域的突破导致了我们今天所知道的 Web 2.0。虽然早期的 Salesforce 等企业和产品是普及该技术的推动者和驱动者，但消费者 Web 2.0 的发展导致了业务空间被遗忘。AppExchange 开发人员网络的目的是让 mashups 和其他应用程序的开发人员更容易在业务领域应用他们的技能，并向 Salesforce 拥有的庞大用户群展示他们的解决方案。

我以前开发过与 Salesforce 集成的应用程序和客户端库，追溯到 2002 年他们的 API 第一次发布时，我对技术和社区自那时以来的发展程度感到惊讶。现在，作为一名开发人员，您可以到开发人员网络注册一个免费的开发人员版帐户，下载您需要的所有开发工具，并了解有关使用他们手头的大量资源和社区为 Salesforce 构建应用程序的所有信息。AppExchange 背后的开发方法是:

*   **想象**–想象一个可以为服务企业客户而构建的应用程序或混搭程序(比如一个费用报告应用程序)
*   **创建**——使用可用的工具包(Ajax、PHP、RubyOnRails、Java、Perl 等)。)和资源(示例代码、在线演示、播客、论坛等。)使用您的免费开发者版帐户创建您的应用程序。
*   **分享**–将您创建的应用程序提交到 AppExchange，让它可用，并从 399，000 名 Salesforce 客户那里点击一下，
*   **成功**–最重要的是，您构建的应用程序为您带来了商机。在这里，您可以与 Salesforce 一起推广您的解决方案并取得成功。

有很多可能性，例如，一个费用报告应用程序可以与 PayPal 集成，与员工结算付款，并向他们发回通知(只是一个想法)。Salesforce 向开发人员提供所有这些，并且不要求任何形式的付款，甚至收入分成。目前 AppExchange 中有许多成功的应用程序，也有许多企业专门在 AppExchange 上开发产品，并从中受益匪浅。一个例子是由比尔·阿普尔顿创立的名为[梦工厂](https://web.archive.org/web/20221003073320/http://www.dreamfactory.com/)的公司。DreamFactory 已经为 AppExchange 发布了许多产品，例如项目管理应用程序 [DreamTeam](https://web.archive.org/web/20221003073320/http://www.dreamfactory.com/dreamteam/index.html) 。在 Techcrunch 上，我们每周都会看到许多新产品涌入非常拥挤的领域(比如在线日历，以及我们最喜欢的 [Ajax 桌面](https://web.archive.org/web/20221003073320/http://www.beta.techcrunch.com/2006/02/28/foldcom-why/)，这些产品没有清晰的收入模式。这些公司中的更多公司应该考虑业务空间以及 Salesforce 通过 AppExchange 为开发者提供了什么。把你的产品放到这个平台上的成本是零，而且巨大的资源量使它变得非常简单。

我还独家看到了即将发布的 AppExchange 中的一些新应用。这些都是我们现在通过 Salesforce 了解的服务混搭。第一个例子是 [Google Maps](https://web.archive.org/web/20221003073320/http://maps.google.com/) mashup，它在地图上显示你的潜在客户或联系人的位置。这个 mashup 是使用令人惊叹的 Ajax toolkit 开发的，它允许应用程序托管在客户端——这意味着 mashup 的开发者不需要主机服务器。另一个令人兴奋的混搭是 Salesforce 与 Skype 的整合。用户所做的是选择他们想呼叫或邀请谁参加会议，然后按下“呼叫”按钮，它将通过 Skype 向这些人拨号并建立会议。您可以将储存在联系人名下的 Skye 用户名与普通电话号码混淆，它会自动处理。

我看到的另一个混搭是 Salesforce 与在线文字处理器 Writely 的集成。启动后，将文档存储在 Salesforce 上的 Salesforce 客户将能够使用 Writely 打开文档，并再次直接保存回 Salesforce。另一个非常简单有效的混搭是与 [Adobe](https://web.archive.org/web/20221003073320/http://www.adobe.com/) 在线 PDF 生成器的集成，允许您从 Salesforce 内的源创建 PDF 并将其保存回您的帐户。

这些 mashups 都有一个共同点——它们是两种优秀服务的完美结合，这两种服务的结合被证明是解决业务用户常见问题的非常有效的解决方案。我可以花很多时间来谈论 AppExchange 和 Salesforce，但我不认为一篇帖子可以公正地对待它们。我希望随着新 AppExchange 开发人员网络和新工具包的发布，越来越多的开发人员会考虑加入 Salesforce 社区，在 Salesforce 上开发产品和业务。我知道我期待着将来查看更多构建在 Salesforce 中的应用程序。
