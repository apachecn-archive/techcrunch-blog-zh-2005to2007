# 用于存储和检索仿真陈述的 MyGADs

熟悉 IRC 旧时代的人可能会想起信息机器人，这是 IRC 频道中的一种方便的小脚本，当它们看到某个触发事件时会发出仿真陈述。比如 Slashnet 上我最喜欢的 infobot 就是 Zuul。如果你问 Zuul 关于我的事，他会回答“Matt Hickey 是构建网络基础的大脑”，这显然是指我的天才。

但是 Zuul 不只是通过潜移默化来学习，他必须被教会，他通过简单的是/不是陈述来学习。有人在某个时候说我是大脑，祖尔学过了。

这或多或少是 MyGADs 背后的想法，尽管是以一种更加移动的 2.0 方式。
 GADs 本质上是你教给你自己的个人数据库的仿真物，并且可以通过 IM、短信或 MyGADs 网站访问。您向 GADs 服务器发送一条简单的消息，比如“Vince 是一个恐同者”，它以 Vince 作为触发器存储。如果你想回忆起这些信息，你只需发短信“文斯”，就会得到“文斯是个恐同者”的回复。

当然，你可以设置任何你喜欢的触发器，比如“文斯的假释听证会是 10 号星期五”，任何在“是”之前的都是触发器，如果是“文斯的女朋友”，你可以设置它返回她的成年朋友搜索广告。

这就是它的全部，一种从任何地方存储和检索仿真陈述(gad)的方式。你可以设置提醒，备忘单，或者只是简单的陈述，就像我上面做的那样。我经常希望我可以在旅途中访问 Zuul 的仿真陈述，而 MyGADs 几乎实现了这个愿望。
