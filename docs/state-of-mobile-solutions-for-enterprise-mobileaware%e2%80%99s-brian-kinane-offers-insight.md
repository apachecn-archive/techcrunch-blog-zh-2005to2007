# 企业移动解决方案的现状:MobileAware 的 Brian Kinane 提供真知灼见

> 原文：<https://web.archive.org/web/http://techcrunch.com/2007/07/27/state-of-mobile-solutions-for-enterprise-mobileaware%e2%80%99s-brian-kinane-offers-insight/>

甚至在移动设备出现之前，一部分员工是移动的，这意味着他们经常出差，并依靠各种工具与总部保持联系。与几年前的频繁出差族相比，如今的移动工作人员更加依赖移动应用。那么，今天(和明天)的移动应用对那些正在移动他们的企业的人来说意味着什么呢？MobileCrunch 与 MobileAware 的执行副总裁兼首席战略官 Brian Kinane 进行了交谈。

Brian Kinane 是 MobileAware 的联合创始人，全面负责 MobileAware 的产品管理和战略业务发展。该角色包括战术和战略产品方向、分析师关系以及与关键移动软件企业和通信基础设施合作伙伴的管理关系。在加入 MobileAware 之前，Brian 曾在包括爱立信、Telia 和 Telenor 在内的一系列行业领导者中担任高级商业咨询、市场营销、业务开发和产品职务，专注于增值移动和融合服务以及相关交付平台。MobileAware.com

**Mobile Crunch:** 您认为企业移动解决方案的现状如何？移动 2.0 提供了让移动企业成为现实的技术吗？

**Brian Kinane:** 移动解决方案传统上专注于提供封闭的专有移动应用程序，为移动工作者编码业务流程。我们现在看到了从这种方法到开放式战略移动平台的转变，这种平台使业务分析师能够为客户和员工快速构建移动业务流程。越来越多的移动设备正在从专门的用例转向支持企业内外更有效的协作。作为这一转变的一部分，移动企业技术正日益与 Web 2.0 和 SOA 概念和技术融合。这使得消费者和业务分析师能够从企业内部可用的在线资源、运营商系统(例如位置、消息)以及通过互联网可用的公共资源(例如谷歌地图)中构建增值移动应用。因此，我们见证了对许多基本移动性构建模块的需求不断增长，包括那些为移动设备提供丰富用户界面并使完整 web 服务架构能够扩展到移动设备的模块。

MobileAware 正在引领实现这种融合移动性和 SOA 的新方法，即移动服务基础设施。通过部署移动服务基础设施，企业业务分析师和开发人员可以快速利用现有的企业服务，为离线和在线移动用例创建复合移动应用程序。移动服务基础设施使移动性成为 SOA 环境的一流成员，现在可以利用全套 SOA 功能，包括 web 服务、企业服务总线、服务注册中心、存储库和组合工具。

**MC:** 阻碍移动企业发展的问题有哪些？看起来一个大问题是让团队中的每个人都使用同一个网络，或者在完美的世界中使用同一部手机。但这些事实上是真正的问题吗？

**BK:** 阻碍企业充分利用移动设备的问题实际上是处理不同网络和设备的复杂性。在网络方面，企业需要解决网络中断、可变的网络 QoS(服务质量)、网络之间的切换和无线安全等问题。就设备而言，企业需要应对多样化的操作系统、外形规格和功能集。移动企业供应商现在通过提供抽象设备和网络复杂性的产品来解决这些问题，从而简化和降低开发和维护移动应用的成本。越来越多的企业不知道底层设备和网络系统。

**MC:** 手机的更新速度比电脑甚至 PDA 都要快，那么这将如何影响企业对移动设备的利用呢？

**BK:** 我们预计企业在应用服务方面将越来越多地支持更广泛的设备，以便他们除了支持现场工作人员之外，还能有效地与客户和高管互动。因此，企业将定义策略并实现一个软件层，该软件层将以不可知的方式跨设备和网络运行。企业将寻求部署将 IT 系统与设备/网络多样性隔离开来的基础设施，并将实施移动设备管理策略来支持不同的设备利益相关方。移动性将越来越多地被定义为与最终用户和业务流程需求相关，而不是与底层移动技术相关。

**MC:**mobile ware 真的能减少移动企业员工可能需要携带的设备数量吗，或者说，这将需要携带另一台设备吗？

**BK:** 通过使用 MobileAware 的技术，单个移动设备可以支持多种应用用例，从移动 web 访问到复杂的客户端驻留移动 ERP 应用。然而，MobileAware 相信根据终端用户的设备偏好为他们提供选择。MobileAware 的技术使企业能够支持最终用户的选择，同时为企业 IT 系统和应用程序提供统一和通用的移动基础设施。

**主持人:**您最近见过哪些客户部署？公司如何适应移动应用程序能为他们做什么？

**BK:** 从金融服务到电信、旅游/物流和政府等多个行业，我们见证了移动客户和员工应用的部署，这些应用在短时间内实现了非常可观的投资回报。我们目前看到了各种各样的业务场景，从将联络中心和企业门户扩展到移动设备的组织，到将核心内部业务流程扩展到移动现场员工的组织。企业不再依赖供应商来定义移动业务流程，而是直接将其业务流程工具与移动技术相结合，以创建全新的移动业务流程，这些流程可以根据最终用户的响应进行优化。最后，我们越来越多地看到企业从一组通用的业务资源中部署内部面向员工的移动应用程序和面向客户的移动应用程序。这方面的例子包括为客户提供移动值机服务，为旅游和运输部门的员工提供移动排班系统。最近，我们看到了移动客户端/服务器和基于移动浏览器的银行业务的重大更新。
