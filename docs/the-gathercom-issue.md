# Gather.com 问题 TechCrunch

> 原文：<https://web.archive.org/web/http://www.techcrunch.com:80/2006/01/16/the-gathercom-issue/>

“新闻 2.0”网站的发展速度快得我都跟不上了。[聚](https://web.archive.org/web/20221130212831/http://www.gather.com/)是最近的。

Gather 是一个社交新闻网站，允许用户提交内容，并根据网站上的活动产生收入分成。他们[夸耀](https://web.archive.org/web/20221130212831/http://www.paidcontent.org/pc/arch/2006_01_14.shtml#053149)在两轮投资中获得了 700 万美元的风险投资。

这是好消息。

我通常不会发关于我不喜欢的公司的帖子，除非我能有建设性。我将指出 Gather.com 的一些弱点。这些只是我的看法，但我认为这里有一些严重的缺陷。

首先，网站设计很差，非常杂乱，没有清楚地说明他们在做什么。

第二， [Newsvine](https://web.archive.org/web/20221130212831/http://www.beta.techcrunch.com/2005/11/09/newvine-to-enter-social-news-ranks/) (关于他们的早期帖子)，仍然处于私人测试阶段，已经显示出将新闻条目与用户评论相结合的力量。这是一种将边缘内容与用户交互相结合的显而易见的方式，并证明了作为中间人的价值。Gather 不会这样做。

第三，收益分成行不通。史蒂夫鲁贝尔说出了原因——市场已经饱和(尽管与史蒂夫不同，我为他们至少尝试直接获得广告客户并对抗谷歌可笑的未公开收入分成比例而喝彩)。我的主要问题是，我认为这不足以吸引用户的注意力。

第四，Gather 鼓励对新闻条目进行标记，但却有一个严格的目录分类法(这意味着最终他们不信任标记来创建他们的目录)。坏主意。跟着标签走，放下分类法，看看会有什么发展。

[博主们](https://web.archive.org/web/20221130212831/http://tech.memeorandum.com/060115/p13#a060115p13)一般都会给聚集一个大拇指下来。马修·英格拉姆在谈论所有竞争对手方面做得特别好。然而，他遗漏了资金雄厚的 Inform.com，这是另一个在产品方向上苦苦挣扎的服务，但它显然建设性地接受了批评。
