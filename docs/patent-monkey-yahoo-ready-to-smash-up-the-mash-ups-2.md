# 专利猴:雅虎准备好粉碎混搭了吗

也是我们知识产权审查的重点…

雅虎获得了一项专利，这项专利有可能影响许多类似公司的业务发展和未来，包括[谷歌](https://web.archive.org/web/20160307101813/http://www.google.com/)、[博客](https://web.archive.org/web/20160307101813/http://www.bloglines.com/)、[网络视频](https://web.archive.org/web/20160307101813/http://www.netvibes.com/)、[罗约](https://web.archive.org/web/20160307101813/http://www.rojo.com/)、 [Dapper](https://web.archive.org/web/20160307101813/http://www.dappit.com/) 、 [Pageflakes](https://web.archive.org/web/20160307101813/http://www.pageflakes.com/) 、 [NewsGator](https://web.archive.org/web/20160307101813/https://www.newsgator.com/ngs/si.aspx) 、 [LifeIO](https://web.archive.org/web/20160307101813/http://lifeio.com/) 、 [as](https://web.archive.org/web/20160307101813/http://www.24eyes.com/) [以及](https://web.archive.org/web/20160307101813/http://blorq.com/Login.aspx?ReturnUrl=%2fDefault.aspx) [as](https://web.archive.org/web/20160307101813/http://www.bayesnews.xpg.com.br/) [萌芽](https://web.archive.org/web/20160307101813/http://www.thefreedictionary.com/) [网页](https://web.archive.org/web/20160307101813/http://www.yourlivewire.net/) [2.0](https://web.archive.org/web/20160307101813/http://www.newsalloy.com/) [新闻](https://web.archive.org/web/20160307101813/http://onelurv.com/) [聚合](https://web.archive.org/web/20160307101813/http://www.feedshow.com/home) [网站](https://web.archive.org/web/20160307101813/http://www.aggcompare.com/)，甚至[一个还没有真正](https://web.archive.org/web/20160307101813/http://wwww.teqlo.com/)[上线的](https://web.archive.org/web/20160307101813/http://earlystagevc.typepad.com/earlystagevc/2007/02/teqlo_a_preview.html)。

1997 年，雅虎正在开发一项核心服务:用户可定制的网页。雅虎的发展在很大程度上决定了它是谁。在此期间，它提交了一份涵盖这一核心概念的申请，这导致了[一项“动态页面生成器”专利](https://web.archive.org/web/20160307101813/http://www.patentmonkey.com/PM/PatentID/5983227.aspx)，该专利涵盖了保存模板和向用户提供实时信息。[另一项“动态页面生成器”专利](https://web.archive.org/web/20160307101813/http://www.patentmonkey.com/PM/PatentID/7171414.aspx)刚刚发布敲定。这里有一个雅虎新发明的专利声明的可靠例子:

有些人说专利申请越长，专利就越没有价值。在这种情况下，我们认为雅虎可能拥有强大的资产。正如我们已经注意到的，在过去的两年中，许多初创公司已经开始利用 RSS 和 AJAX 来改进雅虎的产品。

将索赔与我们提到的 web 2.0 服务联系起来是评估潜在侵权的非常重要的一步。我们认为，这项专利的声明需要满足两个关键要素，才能潜在地“阅读”无数 web 2.0 公司的服务。首先，用户有一个定制的模板，它保存在主机服务器上。第二，实时信息存储在主机服务器上，并通过模板传递给用户。对于大多数新闻聚合器来说，AJAX 很大程度上定义了用户定制，然后是一个保存的模板。在研究实时信息是如何传递的时候， [Mozilla 说](https://web.archive.org/web/20160307101813/http://developer.mozilla.org/en/docs/RSS:Getting_Started:How_RSS_Works) : *“一个桌面 RSS 聚合器将位于 RSS 聚合的客户端。基于 web 的 RSS 聚合器将位于 RSS 聚合的客户端。”*因此，本地存储的实时信息的第二个关键要素似乎也适用。

雅虎主张其专利权的潜力是一个很难不同意的例子:1。雅虎是用户自定义网络体验的先驱；第二。雅虎今天仍在使用这一核心功能，并试图从中受益。随着业绩停滞不前，投资者对市值 410 亿美元的公司管理层的压力肯定会增加。

雅虎可能已经拿到了一张金奖券，但真正的问题是，“它将如何被使用？”
