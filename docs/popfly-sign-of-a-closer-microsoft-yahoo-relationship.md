# Popfly:微软和雅虎关系更密切的迹象？

[![](img/686390d1f7c400d6b5764bf53c7ad604.png)](https://web.archive.org/web/20221006170221/http://www.popfly.ms/) 尽管本月早些时候有铺天盖地的猜测，微软仍未收购雅虎。然而，当时淹没在众多报道中的猜测是，作为“除谷歌之外的任何人”集团的一部分，两家公司可能会形成更强大的关系。

微软 [Popfly](https://web.archive.org/web/20221006170221/http://www.beta.techcrunch.com/2007/05/18/microsoft-launches-popfly-mashup-app-creator-built-on-silverlight/) 可以展示雅虎和微软之间新关系的一部分。

mashup 和 widget 创建者 Popfly 的私人 alpha 测试于 5 月 18 日宣布，并被定位为雅虎 Pipes 的竞争对手。人们期望这些种类的服务集中在各自的创造者的产品上。Popfly 不包括一个与任何谷歌财产相关的预设“块”,但提供了一系列雅虎块，包括雅虎新闻、流量、回答和雅虎拥有的 Flickr 和即将推出的块。

由于微软缺乏对某些垂直行业的本土报道，雅虎可能会提供一些屏蔽，但雅虎新闻与 MSN 新闻源、Flickr 以及我的 Live Spaces 照片一起提供。

如果微软的目标是通过 Popfly 提供一个更具包容性的产品，为什么没有 Google blocks？有许多谷歌相关产品可以改善 Popfly 上的混搭体验。很难相信排除所有谷歌产品不是故意的。相反，包含与微软产品竞争的雅虎相关模块肯定是有人决定的，这样做的目的是有问题的:为什么是雅虎，为什么是现在？

所有这些都不是从 Popfly 中拿走的，它看起来像一个伟大的服务，然而即使是从小事中，人们也可以找到隐藏的意义。
