# 英国活动日历| TechCrunch

> 原文：<https://web.archive.org/web/http://techcrunch.com/2006/10/24/techcrunch-uk-event-calendar/>

正如我承诺的那样，我已经采纳了我上一篇关于这个主题的帖子的反馈，并研究了围绕维护 TechCrunch 英国活动日历的各种想法。最简单的方法是在 TechCrunch UK 上创建一个 Wiki，但这也意味着每个人都必须到这个网站来更新他们的活动信息，这对我来说似乎有点不可思议。当然，人们可以简单地将他们的活动信息通过电子邮件发送给我，我可以张贴出来，就像我过去一直做的那样，但这既耗时又相当乏味——只是因为我喜欢以 hcalendar 微格式张贴它们。

[![gc_button1_en-gb.gif](img/61808bf554696a6310a60e2f35662602.png)](https://web.archive.org/web/20141102210124/http://www.google.com/calendar/render?cid=61mscumt1og7ferbn9inlajh40%40group.calendar.google.com)

下一个选择是在这个博客上放置一个谷歌日历按钮，允许人们订阅 TechCrunch 的谷歌日历。虽然我已经将此日历共享给“每个人”阅读，但问题是它再次迫使人们将他们的活动添加到此日历中，此外，我仍然需要手动为各种活动组织者提供日历的“贡献者”权限。

伊恩·福雷斯特(Ian Forrester)基于这一共同原则友好地创建了一个名为“伦敦社交活动”的谷歌日历，并让十几个在伦敦定期举办活动的人成为“贡献者”，但这个列表绝不是全面的，所以它再次依赖于某人，即我来定期更新贡献者列表。

因此，我最终决定 TechCrunch 英国的最佳选择是使用[Upcoming.org](https://web.archive.org/web/20141102210124/http://upcoming.org/user/2452/),因为许多举办活动的人已经在那里列出了这些活动，这意味着任何想要创建新活动的人都可以自由地这样做，不需要我干预。

所以我个人所做的是订阅一些在英国列出的事件，这个列表现在出现在这个博客的右边。我正在玩一个 WordPress 插件，它将允许“我即将举办的活动”页面在英国 TechCrunch 上完整显示。与此同时，如果方便的话，你也可以选择[订阅](https://web.archive.org/web/20141102210124/http://upcoming.org/syndicate/v2/my_events/2452)我的近期活动列表。我也喜欢 Upcoming.org，因为所有创建的事件都是微格式，这将很快证明非常有用。

此外，我创建了一个 [TechCrunch 英国小组](https://web.archive.org/web/20141102210124/http://upcoming.org/group/2358/)，你可能想加入？我知道已经有一个我加入的非常活跃的[伦敦极客群](https://web.archive.org/web/20141102210124/http://upcoming.org/group/214/)，但是我觉得 TechCrunch 应该代表整个英国，所以我决定为每个人创建一个 TechCrunch 群。因此，如果您正在创建一个新的活动，您可能还想向这个新的 TechCrunch 群组发送一个群组通知？我想知道我是否可以在 TechCrunch UK 中嵌套组？

最后，我还想问一下，如果您正在创建一个新的活动，并希望 TechCrunch UK 在这里报道它，您可以使用添加标签“ **TCUK** ”吗？这样我可以更快更容易地找到您的活动。我发现使用 Upcoming.org 的另一个好处是，你还可以将创建的任何事件添加到你选择的个人日历中——谷歌、Outlook、我的雅虎、ical(苹果)等。并使用雅虎或谷歌地图添加地图。

我希望这能起作用，并对每个人都有帮助？非常感谢所有反馈。
