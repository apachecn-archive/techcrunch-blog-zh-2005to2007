# 简介:垂直飞跃会议——技术危机

> 原文：<https://web.archive.org/web/http://www.techcrunch.com:80/2005/06/30/profile-vertical-leap-conference/>

**事件:** [垂直跳跃](https://web.archive.org/web/20201205120946/https://www.socialtext.net/vertical/index.cgi)

![](img/201944ca499c8435f5502c1cea148621.png)

**日期:**2005 年 6 月 28 日星期二，上午 9:00-下午 5:30

**是什么？**

垂直飞跃是本周早些时候在硅谷中心举行的为期一天的特别会议，由杰夫·克拉维尔和戴夫·麦克卢尔主持。用他们自己的话来说，*“一批全新的专注于垂直搜索的风险投资初创公司开始进入这个领域并引起注意。搜索正在向垂直方向发展——垂直搜索引擎可用于本地企业、产品&购物搜索、旅游搜索、博客&新闻搜索以及分类&工作列表搜索，每月都有更多针对特定领域的搜索引擎发布。”*

很明显，这次活动的重点是搜索，web 2.0 主题在整个事件中非常普遍。该事件在博客上写得很好，我们已经链接到下面的许多帖子。

所有的部分都很有趣，组织得很好。当然，一个关键的重点是在这个领域的投资，以及在利基搜索领域占据主导地位所需的技术和客户方法。

从我们的角度来看，最有趣的部分是新闻/博客搜索。小组讨论由 Om Malik 主持(一开始由 Steve Gillmor 主持，因为 Om 迟到了)。在我们看来，当从 web 2.0 的角度思考他们的公司时，小组成员的座位是象征性的。

从左到右依次是克里斯·托尔斯(Topix.net)、吉姆·皮特科(此外)、斯科特·拉夫(feedster)、坦泰克·切利克(technorati)和奥姆·马利克(Om Malik)。从讨论中很清楚谁“得到”了 web 2.0，谁没有。从左到右，小组成员更加关注 web 2.0，其中 om 达到了顶峰(我在小组讨论后无意中听到一位观众的评论，说“他们在 Om 博客上花的时间比其他所有网站加起来的时间都多”)。:-)

Om 一直在问其他人无法回答的关于用户界面、搜索相关性和标签的问题。处于 web 1.0 一端的 Chris 并不热衷于 RSS 和标签，他反复声明 Topix.net 在决定将内容放在何处时会考虑“哇”的因素。当被问及搜索相关性时，克里斯说:“人们想听的是谋杀和破产，而不是公司赚钱。”。斯科特和坦提克只是摇摇头，大多数观众也是如此。Om 把目光转向观众，轻声笑了起来。

也就是说，topix.net 是一个很棒的网站，在筛选了他们的 250 个 RSS 源后，我找到了一个很棒的并订阅了。

互联网正在发生根本性的转变，一些人开始使用这种新语言。其他人只是听到胡言乱语。

总的来说，一个精彩的会议，了不起的网络活动，以及大量的咖啡全天供应。感谢杰夫和戴夫举办了一场精彩的会议。
