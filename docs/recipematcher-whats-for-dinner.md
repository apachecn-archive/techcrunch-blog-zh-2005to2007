# RecipeMatcher:晚餐吃什么？

 [](https://web.archive.org/web/20221209123742/http://www.recipematcher.com/) 圣地亚哥 [RecipeMatcher](https://web.archive.org/web/20221209123742/http://www.recipematcher.com/) 是一个用户生成的食谱网站，有一个很酷的搜索概念:告诉它你的厨房里有什么，它会推荐食谱。它还会建议相近的搭配，并告诉你一顿完整的晚餐还需要什么。只需点击一下，你就可以将它添加到可打印的购物清单中。

该网站设计良好，包括一些 Flash 组件，尽管也有一些问题。有明显的地方可以从 Ajax 中受益(比如食谱的评分功能)，尽管有 favorites 部分，我还是找不到让食谱成为 favorited 的方法。我还感觉这东西被托管在某人车库的服务器上。尽管如此，就像[提取饮料](https://web.archive.org/web/20221209123742/http://www.beta.techcrunch.com/2006/01/07/extratasty-have-a-drink/)一样，对于喜欢烹饪的人来说，这是一个非常有用的网站。

如果你对此感兴趣，可以与 [AllRecipes](https://web.archive.org/web/20221209123742/http://allrecipes.com/) 、 [FoodNetwork](https://web.archive.org/web/20221209123742/http://www.foodnetwork.com/) 和[Google Recipes](https://web.archive.org/web/20221209123742/http://www.google.com/base/search?a_n0=recipes&scoring=ld&a_y0=9&a_s0=0&a_r=1)(Google Base 的一部分)进行比较，它们都允许按成分进行搜索。到目前为止，似乎只有 RecipeMaker 允许你将食材储存在你的储藏室中，尽管很多人会发现这很有用。

今天下午，我和创始人 Mike Sasaki 聊了几分钟。迈克用几千美元雇来的帮手创建了这个网站，并继续在一家律师事务所工作。

**更新:** [Snacksby](https://web.archive.org/web/20221209123742/http://www.snacksby.com/) (弗吉尼亚州阿灵顿)是一个非常相似的网站，刚刚给我发了邮件。它有更好的界面，更稳定。如果你正在寻找这种服务，也可以去看看。
