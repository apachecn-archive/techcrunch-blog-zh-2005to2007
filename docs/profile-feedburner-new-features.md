**有什么新消息？**

上周四，Feedburner 举办了一个“黑客马拉松”日(见 [Joe Kraus on hackathons](https://web.archive.org/web/20220925022734/http://bnoopy.typepad.com/bnoopy/2005/05/keeping_innovat.html) )，工程师们放弃他们的项目一天，致力于可以在一天内设计、实现和测试的新功能。

他们想出了七个新功能([链接](https://web.archive.org/web/20220925022734/http://www.burningdoor.com/feedburner/archives/001299.html)):

–**Ping 服务器**–提醒 FeedBurner 您的博客已经更新，并希望 FeedBurner 立即更新您的 FeedBurner feed，而不是等待正常的 30 分钟自动更新。
–**给你的 Feed 添加地理标签**–给你的 FeedBurner feed 添加地理位置。
–**标题/描述刻录机**–允许您在不更改源提要或博客工具的情况下更改提要的标题和/或描述。
–**feed medic**–通知您 FeedBurner 在定期处理您的源 feed 时遇到的任何错误。
–**Javascript 发行量报价器**–启用了感知 API，允许你将每日报价器放在你的博客上，显示你的发行量是上升还是下降。
–**image cast**–对 SmartCast 服务的增强，超越音频和视频外壳。
–**stats tracker Dashboard Widget(仅适用于 Mac)**–使用适用于 Mac OS X Dashboard 的全新 StatsTracker widget 查看您的订阅源的传播情况。

我们最喜欢的是用于快速更新的新 ping 服务器(weblogs.com ping 服务器将很快在这里进行介绍)，以及 circulation ticker(放在您博客上的整洁工具)。当 Keith 去度假试用 statstracker 小工具时，我正试图找到一台 mac 电脑。
