# PayPerPost 病毒传播 TechCrunch

> 原文：<https://web.archive.org/web/http://www.techcrunch.com:80/2006/10/12/the-payperpost-virus-spreads>

两项类似于备受争议的 [PayPerPost](https://web.archive.org/web/20221010074537/http://www.beta.techcrunch.com/2006/06/30/payperpostcom-offers-to-buy-your-soul/) 的新服务在过去几天宣布推出: [ReviewMe](https://web.archive.org/web/20221010074537/http://www.reviewme.com/) 和 [CreamAid](https://web.archive.org/web/20221010074537/http://www.creamaid.com/) 。PayPerPost 是一个广告商向博客写手支付产品费用的市场，最近宣布了一轮 300 万美元的风险融资，引起了额外的关注。

PayPerPost 模式让人想起了音乐行业的贿赂行为，这是联邦通信委员会和州检察长仍在试图消除或控制的事情。鉴于博客圈的分布式和未经许可的性质，控制博客作者的报酬将会困难得多。

我们对这些付费服务的立场很明确:它们是博客圈规模和影响力增长的自然结果，但它们破坏了整个生态系统的可信度，误导了读者。
 [](https://web.archive.org/web/20221010074537/http://www.reviewme.com/) [ReviewMe](https://web.archive.org/web/20221010074537/http://www.reviewme.com/)是以文字链接广告为后盾，长期赞助 TechCrunch。它还没有发布，但是今天早些时候已经宣布了。像 PayPerPost 一样，ReviewMe 是一个市场，允许广告商向博客写手付费，让他们写自己的产品。然而，在商业模式上有一些显著的不同。PayPerPost 上的广告商设定了一个单一的费用，支付给所有博客，无论其规模大小，ReviewMe 使用一种基于 Alexa、Technorati 和其他统计数据的算法来确定博客的重要性，并根据计算结果对每个博客收取不同的费用。值得称赞的是，ReviewMe 要求博主披露他们因帖子而获得的报酬，广告商不能要求正面帖子(PayPerPost 使披露成为可选，广告商可以要求正面帖子)。

虽然我们赞赏 ReviewMe 要求披露信息并禁止广告商要求正面帖子的事实，但我们仍然认为付钱给博客写手来写产品是一个非常糟糕的主意。坦率地说，我们对我们的赞助商之一推出这种类型的服务感到不高兴，我们已经通知他们，我们将不允许通过 TechCrunch 推广 ReviewMe。

 [](https://web.archive.org/web/20221010074537/http://www.creamaid.com/) [CreamAid](https://web.archive.org/web/20221010074537/http://www.creamaid.com/) 本周早些时候推出。该服务类似于 PayPerPost，但要求博客作者在帖子中包含一个 Flash 小部件，链接到 CreamAid，并显示其他讨论过该产品的博客帖子。似乎并没有要求博主对产品进行正面评价，但网站上的细节却很少。CreamAid 的部分目标似乎是使用这个小部件围绕付费帖子建立一个社交网络。

我希望 PayPerPost 能尽快要求博主披露信息，并消除广告商要求正面评论的能力。很明显，简单地说我们不喜欢这些服务不会让它们消失。风投现在也参与进来了，PayPerPost 雇佣了大量博主，他们愿意攻击任何认为这是个坏主意的人。鉴于政府参与的可能性非常低，类似于消除无线电行业贿赂的努力，我真的不确定能做些什么来扭转这一趋势。最终，个人博客将不得不建立和维护自己的可信度。
