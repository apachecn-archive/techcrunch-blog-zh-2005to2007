# OpenStreetMap 与雅虎的墨水交易！地图

> 原文：<https://web.archive.org/web/http://techcrunch.com/2006/12/05/openstreetmap-inks-deal-with-yahoo-maps/>

在开源地理数据向前迈进的一大步中，Maps 已经批准了与 [OpenStreetMap](https://web.archive.org/web/20140912173848/http://wiki.openstreetmap.org/index.php/Main_Page) 的交易，这增加了打破英国地图数据垄断的可能性。

[Mikel Maron](https://web.archive.org/web/20140912173848/http://brainoff.com/weblog/) ，一位自由网络开发者和 OpenStreetMap 的负责人之一，告诉 TechCrunch UK & I，雅虎的 Scott Reagan 和 Jeremy Kreitler 美国的地图团队已经允许 OpenStreetMap 在服务中使用他们的图像。从本质上说，这意味着他们将能够在他们的编辑器中实现跟踪。地图和衍生向量。据信这是雅虎首次。已经为一个组织以这种方式使用其地图 API 提供了合法的许可。

马龙说与雅虎的交易。意味着将数据叠加到真实地图上的过程将大大加快。

OpenStreetMap 是一个英国项目，旨在创建和提供免费的地理数据，如街道地图，绕过技术、法律和版权限制的雷区，这些限制通常会减慢和阻碍在线地图项目。这些地图是由志愿者制作的，他们只需带着 GPS 设备在一个区域走一圈，然后将捕捉到的数据上传到 OpenStreetMap 的网站上。从那里，该网站使用维基风格的工具，允许任何人编辑和改进地图。

“真的没有其他人尝试以我们这样的粒度收集数据。即使轨迹不准确，在一个区域内也会有多条轨迹。因此，即使是 10 米的差距，随着时间的推移，这种差异将会消除。我们的工作原则类似于维基百科的信任原则，”他说。

传统地图的问题——正如《卫报》最近的[自由我们的数据](https://web.archive.org/web/20140912173848/http://www.freeourdata.org.uk/)活动所强调的——是它们难以置信的限制性，尤其是在欧洲。在英国，主要的权利持有者是政府所有的[军械测量局](https://web.archive.org/web/20140912173848/http://www.ordnancesurvey.co.uk/)，而像 [NavTech](https://web.archive.org/web/20140912173848/http://uk.techcrunch.com/http:www.navteq.com) 和 [TeleAtlas](https://web.archive.org/web/20140912173848/http://www.teleatlas.com/Pub/Home) 这样的私营企业则充当转售商。到目前为止，OpenStreetMap 覆盖了伦敦、伯明翰、曼彻斯特、布里斯托尔、牛津、剑桥等大城市，以及斯德哥尔摩、柏林和巴黎等开源地图爱好者更有可能居住的城市。迄今为止，它一直专注于欧洲，因为地理数据在很大程度上对欧洲的网络开发人员来说是遥不可及的，而它可以通过美国的 Tiger 等数据集免费获得。OpenStreetMap 甚至计划覆盖非洲。

马龙承认，OpenStreetMap 不会像专业调查那样精确，但考虑到在正式的地图项目中改变任何不准确之处需要长达两年的时间，而且大多数不准确之处都不太可能得到纠正，因为所涉及的内容规模巨大，OpenStreetMap 显然填补了一个有价值的空白。“我们无法与英国国家测绘局竞争，但两者之间有一层数据没有被填满，”他说。

这显然引起了私营企业的注意。Steve Coast 也参与了这个项目，他最近从 Multimap.com 获得了资金，以进一步推动项目的工作，并继续进行 CadCorp 整合。

OpenStreetMap 的前景在未来可能会更好。随着明年手机内置 GPS 单元的广泛引入，能够通过手机上的一些简单的客户端软件将追踪地图数据被动上传到 OpenStreetMap 可能会成为现实。

马龙表示，OpenStreetMap 甚至可以将在一个区域内走动以获取数据的过程变成一个游戏，这带来了许多有趣的可能性。
