# Teamslide web 演示开始托管

 [](https://web.archive.org/web/20230302075828/http://www.teamslide.com/) [Teamslide](https://web.archive.org/web/20230302075828/http://teamslide.com/) 是一项让演示者在打电话时向观众展示在线幻灯片的服务。它类似于 WebEx，但用于幻灯片，而不是完整的实时屏幕共享。有一个单击转换工具，可以使 PowerPoint 幻灯片在 TeamSlide 中可用。Mike Arrington [在 4 月份推出这项服务时回顾了这项服务](https://web.archive.org/web/20230302075828/https://techcrunch.com/2006/04/24/teamslide-could-disrupt-webex/)并表示，非常低的成本可能会威胁到 WebEx 等公司，但 TeamSlide 需要在服务器端安装软件这一事实是一个主要问题。那个问题已经不存在了。尽管 Teamslide 在过去几周就已经推出，但它今天才正式宣布推出托管解决方案。

这个空间的其他玩家还有 [ZohoShow](https://web.archive.org/web/20230302075828/http://zohoshow.com/) 和 [Thumbstacks](https://web.archive.org/web/20230302075828/http://thumbstacks.com/) (我们对这两个的比较)。另见 [DimDim](https://web.archive.org/web/20230302075828/https://techcrunch.com/tag/dimdim) ，这是 Alpha 中的一个开源非托管网络会议服务，期待下周另一个免费网络会议服务的推出。Teamslide 专注于一项任务，有一个清晰的业务模型，比上述任何其他选项都更加坚实、稳定。

大约每月 10 美元，相对于您的订阅时间，上下浮动几美元，Teamslide 托管用户可以获得以下功能:

*   上传的演示文稿数量不限，文件大小限制为 20MB。竞争对手 ZohoShow 将文件限制在 5MB。
*   总共高达 50MB 的存储空间用于您的演示。
*   用矩形标记或箭头突出显示幻灯片部分内容的能力。ZohoShow 刚刚为他们的演示添加了一些 Flash 绘图工具。
*   最多可容纳 10 人的演示。

由于 Teamslide Hosted 是基于浏览器的(PHP 和 ajax)，因此不存在像许多网络会议服务那样的跨平台兼容性问题。在某些方面，像 Teamslide 一样保持简单是一个真正的优势。不能共享桌面，也不能做现场演示，但至少很好用。我经常参加网络会议，这些会议可能已经被一个轻量级的幻灯片和一个电话所取代。
