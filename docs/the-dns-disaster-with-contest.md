# DNS 灾难(带竞赛！)

我成为贝尔南方 DSL 用户已经有 10 年了。这有时是一种混乱的关系。开始的时候，我玩了很多游戏，也有一些问题，但是在一年的抱怨之后，一切都平衡了。从那以后一直很棒。

这些年来，我一直在升级到现有的最高套餐。我目前订阅了 DSL Xtreme 6.0 包。除了这里或那里的短暂中断，它一直是一个无故障的体验——直到大约三周前。


一天早上(好吧，下午)我醒来，希望像往常一样在线。有些不对劲。花了 pages 分钟访问。没问题，我想。重置网络设备，但问题仍然很明显。碰巧的是，我计划那天晚些时候飞往洛杉矶，所以我忘记了这件事，起飞了。

向前跳一周。我回到家也遇到了同样的问题。慌乱之中，我给& T 的~~南方贝尔~~打电话，告诉他们出了问题。“我们这边没什么问题，”技术顾问说。我知道我这边没什么问题，但我还是默认了让一名技术人员第二天来我家。

生产线技术人员来了，他当然没有发现任何问题。然而，他用 Cat-5 替换了一些老化的成对品系。带宽略有改善，但访问网站的问题依然存在。

我给技术支持打电话，告诉他们问题仍然存在。他们仍然坚持这是我的责任。我经历了一些困难(更多神奇的修复一切的解决方案，重启)，变得越来越好战。我建议他们检查他们的 DNS 服务器，这被认为是一个荒谬的假设。争吵继续着，我变得愤怒了。

在等待的时候，听了第 100 遍最难以忍受的肯尼·基废话，我想起了我最近看到的一些东西:[OpenDNS.com](https://web.archive.org/web/20210303002646/http://www.opendns.com/)。经过一段时间的等待，我设法进入了网站。切换到 OpenDNS 服务器，就像变魔术一样，一切又恢复了正常。

与此同时，技术 gimp 继续坚持美国电话电报公司 DNS 服务器没有任何问题。这到底是怎么回事？对我来说，这是一个非常透明的问题。切换 DNS，一切正常，肯定是 DNS。对吗？我错了吗？有什么我没看到的吗？或许是阴谋？

我对此非常生气。在拥有南方贝尔多年后，我是一个相对快乐的顾客。然而，在美国电话电报公司接手的几个月内，我经历了一件非常不幸的事情。这不是好兆头。

我对 OpenDNS 也非常满意。这是一项了不起的服务，我应该为此得到很多支持。特性集是优秀的，访问率是一流的。我建议你们都立即检查一下。
