# PrefPass 使匿名个性化变得容易

 [](https://web.archive.org/web/20220626135302/http://prefpass.com/) [PrefPass](https://web.archive.org/web/20220626135302/http://prefpass.com/) 是一个新项目，旨在为用户提供各种网站的个性化体验，而无需放弃个人信息或通过 tedius 注册表格。它还能让网站出版商提供更有针对性的内容和广告，而不会打扰他们的读者。这是一个非常酷的想法；如果你喜欢[BugMeNot.com](https://web.archive.org/web/20220626135302/http://bugmenot.com/)，一个将虚拟账户汇集到需要账户注册的新闻网站的网站，那么你会真的喜欢 [PrefPass](https://web.archive.org/web/20220626135302/http://prefpass.com/) 。对每个参与的人来说都是双赢。

PrefPass 是由旧金山的 Adam Marsh 开发的，目前处于私人测试阶段。您可以在主页上申请一个测试帐户。

用户提供他们关联的网站的 URL，比如他们的博客或他们喜欢阅读的网站。在这些网站上找到的类别和标签，可能也来自其他来源，被作为“偏好”或你感兴趣的东西抓取。当您访问参与 PrefPass 的网站时，您可以单击该网站上的按钮授予该网站查看您的偏好或兴趣的通行证。然后，该网站可以提供各种定制的内容，从该网站的推荐博客帖子，到整个网络的推荐博客帖子，以及通过 Yahoo！或者，当然，根据你的个人喜好做广告。

站点发布者会收到报告，详细说明其站点中已授予 PrefPass 的访问者的兴趣。这里的图片来自我的个人博客[Marshallk.com](https://web.archive.org/web/20220626135302/http://marshallk.com/)的侧边栏，列出了在我表明自己对 TechCrunch 感兴趣并允许 MarshallK.com 查看我的兴趣后向我推荐的帖子。

在该服务离开测试版之前，有一些小事情是很好的，比如只有在用户授予网站 PrefPass 后，推荐帖子的列表才会出现，现在没有登录的网站访问者可以看到所有最新的帖子。如果能对这些小部件中出现的项目数量有更多的控制，那就更好了。如果这里的搜索/RSS/javascript 播放能像使用彼得·库珀的 [FeedDigest](https://web.archive.org/web/20220626135302/http://feeddigest.com/) 一样流畅，那就太好了。

这是一个相当聪明的系统。javascript 尊重每个网站的 CSS，有各种各样的小部件可供出版商使用，基本思想似乎真的对读者/出版商双方都有利。在系统证明你是一个有真实兴趣的真实的人的同时，你的匿名性得以保持，这一事实很棒。Prefpass 让读者可以轻松地、几乎是愉快地允许出版商提供个性化的、有针对性的内容。

这项服务的商业模式似乎还没有确定，但显然不缺乏方向。如果创收部分做得很好，而且这个系统可以像病毒一样传播，我认为 PrefPass 可能会是一个大赢家。BugMeNot 对网站发布者来说是个小麻烦，对用户来说也不是很方便——但是让 [PrefPass](https://web.archive.org/web/20220626135302/http://prefpass.com/) 传播开来符合每个人的利益。
