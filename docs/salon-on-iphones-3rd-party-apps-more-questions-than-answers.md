沙龙的法尔哈德·曼朱发表了一篇相当有说服力的分析文章，分析了 iPhone 和 OS X 现在将会支持的 Web 2.0/Safari/No SDK 风格的编程。他基本上问了所有相关的问题，但可能要到 10 月份才能得到答案:

> 你能在 iPhone 上运行 Skype 吗？开发者能为 iPhone 开发一个与 iTunes 竞争的点唱机应用吗？照片编辑程序怎么样？那么 Safari 上的 Firefox 呢？

虽然 Safari 上的 Firefox 有点牵强，但我完全同意。这是阻碍了 iPhone 的创新，还是让它向全新的受众开放？这也提出了一个很有意思的问题。这个工具箱会出现在*所有*版本的 Safari 上吗？如果会，你能跨平台运行你的 Safari 应用程序吗，就像你在 [XUL](https://web.archive.org/web/20160421220602/http://www.xulplanet.com/) 上做的那样。也许编程语言只是某种 Ajax-y XML 之类的东西？

这些都是重要的问题，因为正如我们在 Boot Camp 看到的，苹果知道它需要进入微软。在多年扮演次要角色后，他们有足够的思想份额来说服人们将他们公认的次等浏览器安装到 Windows boxes 上，即使考虑到所有流行和重要的竞争对手。由于速度和兼容性问题，我停止使用 Safari，但如果我能从 iPhone 中获得某种奖励，我会再使用它。奖金是多少？我头疼。
