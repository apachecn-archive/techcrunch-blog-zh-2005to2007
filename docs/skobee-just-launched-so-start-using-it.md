# Skobee 刚刚推出——开始使用吧——TechCrunch

> 原文：<https://web.archive.org/web/http://www.techcrunch.com/2006/03/09/skobee-just-launched-so-start-using-it/>

 [](https://web.archive.org/web/20220814191231/http://www.skobee.com/) 自从我第一次在斯坦福最近的一次[活动](https://web.archive.org/web/20220814191231/http://www.beta.techcrunch.com/2006/01/28/nine-startups-at-e27-summit/)上看到创始人诺姆·洛温斯基演示以来，我就一直在追踪总部位于旧金山的[斯科比](https://web.archive.org/web/20220814191231/http://www.skobee.com/)。本周早些时候，诺姆带我再次体验了该产品的演示，以唤起我的记忆，他在今晚早些时候打开了这项服务的大门(看起来[戴夫·温纳](https://web.archive.org/web/20220814191231/http://www.scripting.com/2006/03/08.html#When:10:17:48PM)也看到了它)。

如果你用电子邮件和朋友一起计划活动，Skobee 会让你的生活变得更加轻松。

不要通过正常渠道(即时消息、电子邮件、电话等)联系所有你想一起计划活动的朋友。)并试图让事情井井有条，Skobee 有一个非常简单且更好的方法来处理它。创建一个新活动，输入模糊(或明确)的地点和日期/时间信息，并为您希望包括的人添加电子邮件。

然后，Skobee 将向每个人发送电子邮件以及当前的活动信息，如果他们是新用户，他们可以注册该网站(病毒式商业模式——是的)。事情变得有趣起来:这些人可以添加新的参与者，并通过留下自然语言的评论来建议更改日期或地点，例如“周二怎么样，我们在 TechCrunch 见面”。Skobee 将数据结构化，并为每个人更新活动信息。更好的是:参与者可以简单地“回复所有”电子邮件，包括“plans@skobee.com ”,而不是登录网站并留下评论，skobee 将计算出正在讨论的活动，自动将评论添加到活动页面，并相应地更新。

我已经用 Skobee 计划了一个活动(见下面的截屏),上面的所有功能都运行得非常完美。

未来的计划包括一个“派对后”页面，Skobee 将自动从用户那里获取 flickr 图片，并将它们包含在页面上。Noam 表示，这将基于图片元数据时间来了解图片是否适用，尽管我建议他们使用 flickr 标签来代替或补充来自照片的元数据。
