# TagLoops:由标签和提要组成的可再混合的网络电影

 [](https://web.archive.org/web/20210919142022/http://www.tagloops.com/) [TagLoops](https://web.archive.org/web/20210919142022/http://tagloops.com/) 是由希腊 IT 顾问哈里·维卡托斯打造的 alpha stage Flash“电影”创作工具。电影是通过指向其他地方托管的图像、音频或 RSS 源来构建的。这些项目被标记在 TagLoop 中，在这里它们可以被重新排序和分层。最终结果是一个多层多媒体 Flash 电影。其他 TagLoop 用户也可以对这些电影进行混音。对我来说，TagLoops 与标签、提要和 URL 等短暂的东西一起工作，最终让用户创建一个有形的多媒体演示，这真的很有趣。

就像 Michael [昨晚评论的应用程序](https://web.archive.org/web/20210919142022/http://www.beta.techcrunch.com/2006/10/04/introducing-slideshare-power-point-youtube/) [SlideShare](https://web.archive.org/web/20210919142022/http://slideshare.net/) 一样，TagLoops 是制作丰富的在线演示的一种相对简单的方式。这两种方法都不适用于视频，但 TagLoops 的目标是。

这里有一个我制作的的例子，突出了一些我最喜欢的 [GapingVoid](https://web.archive.org/web/20210919142022/http://gapingvoid.com/) 漫画。我在 Del.icio.us 上标记了我最喜欢的内容，然后把它放到 TagLoops 上。如果我在同一个 RSS 订阅源中添加更多卡通，它们将自动添加到我的 FilmLoops 电影中。指向托管音乐的 URL，音乐就会以流媒体形式播放。我的“网络电影”非常简单，但在服务中有很大程度的复杂性。这里是 Vikatos 制作的站点的 TagLoops 风格演示。

这个界面对我来说很难学会使用，但现在我有了信心，我可以很容易地使用这个工具制作更多的电影。这是一种看似简单的制作网络短片的方法。网站上有一个演示帐户，任何人都可以使用它来试用。

当您将 RSS 源放入您的帐户时，您可以自动播放该源中的所有图像或音频，或者只选择一些项目拖放到您的电影中。这是一个相当不稳定的系统。

音频文件可以有效地在 TagLoops 中编辑。视频和 Flash 动画是 TagLoops 的下一个项目类型。

该网站将从一些可用性工作和更清晰的说明中受益匪浅，尽管有一个帮助 wiki 可用。这里有很多，对于一个人的兼职项目来说，这是非常令人印象深刻的。由标签和 feeds 提供的多媒体制作的可再混合的拖放式电影是你能得到的最前沿的东西。行得通吗？只有时间能证明一切，但我可以说，使用它比描述它要容易得多。
