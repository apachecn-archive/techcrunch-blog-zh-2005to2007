# Technorati 发布主要新功能——TechCrunch

> 原文：<https://web.archive.org/web/http://www.techcrunch.com:80/2005/09/01/technorati-releases-major-new-feature/>

戴夫·西弗里[今晚宣布](https://web.archive.org/web/20220527124147/http://www.sifry.com/alerts/archives/000339.html)推出 [Technorati 博客发现者](https://web.archive.org/web/20220527124147/http://technorati.com/help/blogfinder.html)。Technorati 的 Derek Powazek 也在 Blog Finder [上发表了文章。](https://web.archive.org/web/20220527124147/http://www.technorati.com/weblog/2005/09/44.html)

这是什么？这是博客的排名列表。按标签。大卫·波瓦泽克称之为“你能想到的任何话题的迷你前 100 名”。

这是卡特里娜飓风。下面是 [Web 2.0](https://web.archive.org/web/20220527124147/http://www.technorati.com/blogs/web2.0) 。这是 [Skype](https://web.archive.org/web/20220527124147/http://www.technorati.com/blogs/skype) 。

有多好？我觉得它有些问题。标签是出版商创建的，所以出版商会添加大量的标签，并且会有动力为他们的博客设计一个合适的标签。举例来说，我相信只要在 TechCrunch 博客上添加“烹饪”标签，就能让 TechCrunch 成为一个非常受欢迎的烹饪网站。

添加标签有三种方式。根据 Sifry 的数据，Technorati 已经标记了“超过 200 万个博客”，这似乎是准确的。添加标签的另外两种方式是通过 publisher 工具(注意，下面的文本是为添加“web2.0”标签而定制的):

> 快进来。要将您的博客添加到此页面，[声明您的博客](https://web.archive.org/web/20220527124147/http://www.technorati.com/account/#blogs)并将“web2.0”添加到您的博客标签中，或者将[此代码](https://web.archive.org/web/20220527124147/http://www.technorati.com/blogs/web2.0#taglink)放在您的网站上。

然而，Technorati 在选择描述 Techcrunch 的标签方面做得相当不错。我肯定会做出一些改变，但这是一个好的开始。

![](img/fccf8747d8b15f2f9543bfe38b116dba.png)

博客搜索器是 Technorati 上的第三个标签。

乍一看，我实际上认为在这种情况下我更喜欢罗约的方法。罗约的列表使用读者用来描述博客发布的单个帖子的标签。关于的详细信息，请点击这里查看我们的个人资料更新。然而，我确实相信，如果列表有用，它必须是基于主题的。这是解决这个问题的一个有趣的实验。
