# 纽约时报推出 MyTimes，一个弱 RSS 播放

NYTimes.com 推出了一个名为[MyTimes.com](https://web.archive.org/web/20221001084316/http://my.nytimes.com/browse/)[的个性化新闻网站的有限测试版(截图](https://web.archive.org/web/20221001084316/http://www.flickr.com/photos/13736953@N00/tags/mytimes/) via PaidContent)，一个看起来更像 MyYahoo 而不是最近与 [MyUSAToday](https://web.archive.org/web/20221001084316/http://my.usatoday.com/news/myusatoday.aspx) 和 [MyNewsweek](https://web.archive.org/web/20221001084316/http://headlines.newsweek.com/NGWhiteLabel/Sites/NWk/reader.aspx) 的 Newsgator 合作的 RSS 播放器。与这些服务不同，用户似乎不能将推荐源之外的源添加到他们的 MyTimes 页面。如果是这样的话，那真是一种耻辱——我认为，主要媒体公司提供有效品牌的 RSS 阅读器，对默认的 OPML 文件进行编辑控制的做法，比这种对 RSS 的驯服使用要好。(更新:可能不是这样，但我还是不在乎这个服务的格式，原因如下。)

PaidContent [报道](https://web.archive.org/web/20221001084316/http://www.paidcontent.org/nytimess-personalized-service-my-times-debuts-in-limited-beta)my times 目前仅限于 5000 名已经表示出兴趣的用户，但稍后将开放供公众使用。RSS 创始人 Dave Winer 说他想做一个关于如何为 RSS 阅读器设计界面的研讨会。

在开放使用之前，很难确切知道这将如何工作；但是当有限数量的订阅源中只有几个最近的条目可以被浏览时，那么**RSS 的普及是以巨大的功能损失为代价的**。我喜欢 RSS，对这样的事情感觉很矛盾。是的，RSS 支持小部件(甚至 MyTimes 也这么称呼它们),但是它可以做更多的事情。即使[的 MyUSAToday](https://web.archive.org/web/20221001084316/http://my.usatoday.com/news/myusatoday.aspx) 和[的 MyNewsweek](https://web.archive.org/web/20221001084316/http://headlines.newsweek.com/NGWhiteLabel/Sites/NWk/reader.aspx) 网站有点笨拙，不那么漂亮，但它们对于严肃的阅读确实有用。添加您自己的来源的能力是关键，让我对您的社区编辑的平台和洞察力印象深刻。启用新闻河意味着读者可以根据所有来源的最新信息来查看项目，这很关键，因为一旦你选择了来源，那么单个项目出现的时间可能会变得比它们来自哪个来源更重要。将新闻显示为一个由离散积木组成的领域，这是一种与即时世界联系起来的原始方式。小工具对很多事情来说都很棒，但是不要告诉我通过它们来查看整个世界的新闻。

这里有更多关于 Newsgator 对 RSS 的展望。《纽约时报》提供的 RSS 体验比《今日美国》更加乏味，这难道不具有讽刺意味吗？
