# Pandora/Last.fm 混搭更新

[![](img/0aa9fddf0287ecc4788e6eb1505ea73c.png)](https://web.archive.org/web/20221031141214/http://pandorafm.real-ity.com/) 如果你是[潘多拉](https://web.archive.org/web/20221031141214/http://www.crunchbase.com/company/pandora)、 [Last.fm](https://web.archive.org/web/20221031141214/http://www.crunchbase.com/company/last.fm) 或[两者](https://web.archive.org/web/20221031141214/http://www.beta.techcrunch.com/2006/01/23/655/)的粉丝，你会喜欢 Gabe Kangas 的这个[混搭](https://web.archive.org/web/20221031141214/http://pandorafm.real-ity.com/)。我上个月[写了这篇文章](https://web.archive.org/web/20221031141214/http://www.beta.techcrunch.com/2006/03/29/pandora-and-lastfm-togethersort-of/)，当时它还有点烂。现在，Pandora 完全支持这项工作，甚至允许 Gabe 访问 Pandora API 来创建一个更稳定的应用程序。

mashup 基本上是潘多拉播放器，听众可以将潘多拉播放的音乐直接添加到他们的 Last.fm 播放列表中。Last.fm 还显示用户最喜欢的艺术家，并可用于即时生成 Pandora 电台。

潘多拉官方对 mashup 的认可让我感到惊讶的是，它有效地获取了潘多拉的用户数据，并直接发送到 Last.fm。这对他们认识到开放比封闭更好很有好处。
