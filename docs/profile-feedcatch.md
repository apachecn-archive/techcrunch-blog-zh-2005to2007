# 简介–feed catch–TechCrunch

> 原文：<https://web.archive.org/web/http://www.techcrunch.com/2005/08/13/profile-feedcatch/>

## 概述:

感谢 [Nivi](https://web.archive.org/web/20220527121943/http://www.nivi.com/blog/) 给我介绍了 [FeedShake](https://web.archive.org/web/20220527121943/http://www.beta.techcrunch.com/?p=102) 的新 [FeedCatch](https://web.archive.org/web/20220527121943/http://feedcatch.feedshake.com/) feed 存档服务。

FeedCatch 是一个很好的小工具，它允许你无限期地获取任何 feed 和存档帖子(或者如果你设置了参数，就不会这么做)。有许多默认选项，包括:

*   设置 X 篇新文章发布后删除的文章
*   将帖子设置为在 X 小时后删除
*   设置点击链接后要删除的文章

只需几秒钟即可设置一个馈送并选择归档参数。显示标题和一些摘要文本。

请注意，您可以使用 FeedShake 服务从多个提要创建一个合并/排序/过滤的提要，然后使用 FeedCatch 归档所有内容。

看看这个。这项服务是免费的，不需要注册。我们创建的 TechCrunch feed 在这里是。

有一个正在处理的错误报告。希望这将很快变得稳定(因此有用):

> GMT::为了修复一个错误，在此日期之前存储的所有项目都将被删除。这种情况可能会再次发生，所以不要指望 FeedCatch 会在本周存储任何稳定的数据。[链接](https://web.archive.org/web/20220527121943/http://feedcatch.feedshake.com/bugalerts.html)
