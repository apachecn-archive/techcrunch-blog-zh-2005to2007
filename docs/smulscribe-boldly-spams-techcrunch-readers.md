# SimulScribe 大胆地向读者兜售 TechCrunch

几天前，我写了一篇关于 Spinvox 的文章，这是一家将语音邮件转换成文本并通过电子邮件和/或短信发送给你的初创公司。他们刚刚在美国推出，Spinvox 给了我们 100 个账户免费赠送(这项服务相当昂贵)。超过 300 人在评论中回应说他们想要一个。

由于公司自己跟进，我们管理这些账户赠品的最简单方法是要求评论者在评论中留下他们的电子邮件地址。收获的风险是显而易见的，但人们愿意承担，因为这是我们协调事情的唯一简单方法。这是我们过去在成功做过多次[的事情。](https://web.archive.org/web/20211207031432/http://www.beta.techcrunch.com/2006/08/15/finally-almost-allpeers/)

我们没有想到的是，Spinvox 的竞争对手之一 [SimulScribe](https://web.archive.org/web/20211207031432/http://www.simulscribe.com/) 会在没有得到他们许可的情况下，直接收集电子邮件并向他们发送垃圾邮件。但这正是他们所做的。至少一些在评论中留下电子邮件以获得免费 Spinvox 试用的人收到了 SimulScribe 首席执行官詹姆斯·西米诺夫(James Siminoff)主动发来的电子邮件，敦促他们尝试他们的服务。Simulscribe 甚至使用了主题行“TechCrunch 的免费试用”——这是一条误导性的(并且侵犯了商标)信息。

诚然，这是一个大胆的营销举措，但也是一个非常值得质疑的举措。潜在客户在与他们做生意之前会考虑 SimulScribe 对个人信息的态度。就我们而言，我道歉。将来我们会找到一种更好的方式来提供帐户。

致 spin vox——这是一个绝佳的营销机会。你应该给帖子的每个[评论者一个免费账户。他们会是忠实(健谈)的顾客。](https://web.archive.org/web/20211207031432/http://www.beta.techcrunch.com/2007/01/17/spinvox-launching-in-us-we-have-100-accounts-to-give-away-now/)
