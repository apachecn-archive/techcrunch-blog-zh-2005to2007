# 我希望 IE 7 即将发布的十件事——TechCrunch

> 原文：<https://web.archive.org/web/http://www.techcrunch.com:80/2006/10/09/ten-things-i-wish-ie-7-was-about-to-deliver>

与一些报道相反， [Internet Explorer 7](https://web.archive.org/web/20220216114702/http://www.microsoft.com/windows/ie/default.mspx) 可能不会在明天的周二补丁更新中自动向 Windows 用户推出；根据 IE 7 团队博客显示，它将在周四后的某个时间[发布。不管怎样，现在似乎是时候去看看了。IE 的测试版有一些急需的改进。其中大部分似乎受到了更小、更具创新性的浏览器的启发。利用 IE 巨大市场份额的网络效应的更多创新将是有趣的，而不是推出另一种最好的单独为个人用户服务的浏览器。](https://web.archive.org/web/20220216114702/http://blogs.msdn.com/ie/archive/2006/10/09/Last-Expert-Zone-Chat-Before-We-Ship-IE.aspx)

首先，我将提供一个好消息的概述，然后列出我希望 IE 7 包含的 10 件事，以便让我对此感到兴奋。
 **好消息**

IE 7 比以前的版本好看多了，差距很大。如果你还没有看到候选版本 1 的界面，在这篇文章的最后有一个截屏。事实上，它的某些部分看起来很像弗洛克。

它支持选项卡式浏览。标签很容易操作，并且有一个非常好的选项可以在一个屏幕上查看所有标签的预览窗格——非常适合处理标签过载。

您可以在浏览器中订阅 RSS 源；OPML 文件可以轻松导入和导出。完整的提要可以在发布时查看，并且您可以管理提要阅读器的一些细节(例如附件的自动下载和单独安排的刷新)，但是 RSS 的新实现仍然存在一些主要问题。许多人都说 IE7 中的 RSS 将意味着非技术用户中 RSS 阅读的爆炸式增长。我一点也不确定那是不是真的，但稍后我会详细说明。

浏览器的角落里终于有了一个搜索框，还有备受吹捧的反钓鱼控件。你可以很容易地选择在下次打开 IE 时重新打开你当前的标签页。

还有就是 Windows 的集成。NET messenger，但稍后会详细介绍。
 **如果事情不是这样…**

仍然有许多缺点。总的来说，这是一个进步，我很感激如果没有其他事情，标签式浏览将会在世界范围内传播。有些人可能会说，我只是在寻找功能过载，但显然已经添加了许多新功能。我只想看到这个版本中更多更好的实现。

从现实到幻想，我希望世界上最大的浏览器在其最新版本中提供一些东西。
 **1。有用的 RSS 查看**

没有一个“新闻河”选项来查看我的所有提要或一次查看一个文件夹中的所有提要，我还不如直接点击浏览器收藏夹列表。一次点击一个 feed 的选项远远没有达到 RSS 提供的潜力，就目前情况来看，IE 7 中的 feed 视图只是一次查看一个页面的方式，可能有截断的条目，没有页面设计。自动下载附件的选项是这个实现在 RSS 中唯一成功的部分。
 **2。一键 RSS 订阅**

信不信由你，当浏览器在一个页面上检测到一个提要时，你仍然必须查看那个提要，然后选择再次点击订阅。我幻想着有一天我能够查看我所在领域的所有 feeds，但是现在至少让订阅变得容易。
 **3。标记和在线访问收藏夹**

标签很有用。这就是一件物品如何被分类，从而以多种方式被发现。对于大多数人来说，文件夹可能是组织收藏夹或提要的首选方式，但按标签查看的选项会非常有用。同样，当离开我的电脑时，在线访问我的收藏夹和订阅源是每个人都会喜欢的。当你告诉一些人我们称之为社会书签的东西可以让他们从任何电脑上访问他们的书签时，你会看到他们的眼睛亮了起来。
 **4。拖放组织**

如果你要给我文件夹，至少要让我能方便地在里面移动东西。大多数其他在线阅读器现在都支持这一功能。
 **5。提要阅读器中的多媒体**

Bloglines 和其他一些 feed 阅读器现在支持在用户界面中查看视频文件。IE 7 从 feed reader 视图中剥离像嵌入视频这样的文件。
 **6。社会注解**

IE 占据了如此大的市场主导地位，以至于让所有这些用户就他们访问的页面进行交流是有意义的，如果他们愿意的话。从 StubleUpon 获取一个页面，或者尝试为每个 URL 提供一个 wiki 侧边栏。如果 IE 的市场份额能够被用户所利用，而不仅仅是微软获利的一种手段，这将是一件好事。

7。P2P

同样，P2P 在浏览器中也很强大。请看 Opera 的最新版本，它内置了 BitTorrent。我知道——这对于 IE 来说太不现实了，但这难道不合理吗？即使一个 DRM 锁定的 P2P 系统也能更快地下载。
 **8。支持用户控制注意力数据**

参见来自 AttentionTrust.org 的火狐插件。有朝一日，捕捉、利用和选择性曝光我们的在线活动的能力将被视为一项基本的公民权利。我知道这有点异想天开，但如果微软能尽快在这方面采取更多措施，那就太好了。

**9。全跨平台 IM 兼容性**

Windows Live Messenger [宣布与雅虎的 IM 兼容](https://web.archive.org/web/20220216114702/http://www.beta.techcrunch.com/2006/07/12/windows-live-yahoo-im-interoperability-begins-public-tests-today/)。七月有声音的信使。各种 IM 提供商已经宣布了有限数量的其他跨平台兼容性。IE7 中的即时消息似乎甚至不能在雅虎上使用。为什么他们不能结束这场游戏，建立或购买一个跨平台的即时通讯程序呢？这要求过分吗？
 **10。符合标准**

据报道，IE 7 是迄今为止最符合标准的浏览器版本，但如果能抛弃恃强凌弱的态度，至少能完全符合 WC3 标准，那就太好了。取而代之的是，网络得到一个警告，IE7 即将到来，每个人最好测试他们的应用程序，看看它们是否能在新的浏览器中工作。标准创造了一个创新的公共空间，就这么简单。*具体讨论见下面的评论。* 
**10.5 办公在线**

这可能很牵强，甚至没有进入前 10 名，但看看谷歌和 Zoho 这样的初创公司都在做什么。这就是未来。让我通过我的浏览器尽可能地做更多的事情，让我从任何一个物理位置获得自由，让我在适当的地方利用网络效应。

在一个重要的新产品发布的时候，对它的本来面目进行反思可能是不公平的；但是考虑到 IE 的巨大市场应用和网络其他领域令人难以置信的创新，我希望我对这篇文章顶部的好消息给予了足够的认可。底线是 IE 应该一直关注其他人在做什么，但是要比它走得更快。
