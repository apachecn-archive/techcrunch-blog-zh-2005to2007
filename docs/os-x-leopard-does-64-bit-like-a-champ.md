# OS X 豹:64 位像冠军吗

[](https://web.archive.org/web/20210118210257/http://photobucket.com/)

在所有关于可爱的新[聚光灯可视化和花哨的桌面小部件](https://web.archive.org/web/20210118210257/http://www.macrumors.com/pages/2006/06/20060623081820.shtml)的讨论中，只有询问者决定继续讨论 Leopard 的 64 位系统。看来，Vista 将只在高度专业化的操作系统版本中支持 64 位处理，该版本将在软件中模拟 32 位处理。

这一层被称为 Window on Windows 或 WOW，它将运行旧的 XP 应用程序，但不能运行旧的驱动程序。Leopard 开箱后将完全符合 64 位标准，内置非模拟 32 位支持。

> 从 Xcode 2.4 开始，OSX 通用二进制格式已经扩展到在同一文件中支持 PPC 和英特尔处理器的 32 位和 64 位，从而提供了 OSX quad 通用二进制。无论是在操作系统层还是在应用层，用户都不必在处理器架构或 32 位或 64 位处理器之间进行选择，这一切都已经从它们中抽象出来，这是理所应当的。

我知道现在谈论内核仿真还为时过早，但你们当中的黑客会很高兴地知道，你很快就能以内置 XP/Vista 支持的 [Mac Pro](https://web.archive.org/web/20210118210257/http://www.appleinsider.com/article.php?id=1947) 的价格拥有一个完全 64 位的操作系统，你知道，以防万一。
