# Newsgator 发布了 RSS 未来的路线图

 [](https://web.archive.org/web/20220627213856/http://www.newsgator.com/) [Newsgator](https://web.archive.org/web/20220627213856/http://www.newsgator.com/) 和 [Feedburner](https://web.archive.org/web/20220627213856/http://feedburner.com/) 是目前 RSS 领域最活跃的两家公司。当这些公司中的任何一家发表言论时，我都会密切关注。昨天，Newsgator 创始人兼首席技术官 Greg Reinacker(听一听在 [TalkCrunch](https://web.archive.org/web/20220627213856/http://www.talkcrunch.com/2006/04/03/episode-4-battle-of-the-online-feed-readers/) 上对 Greg Reinacker 和其他 feed 阅读器的高管的采访)发布了每个对 RSS 的未来感兴趣的人都应该注意的东西——该公司近期未来的大[路线图](https://web.archive.org/web/20220627213856/http://www.rassoc.com/gregr/weblog/archive.aspx?post=811)。

根据 Mike 在 CrunchNotes 上的帖子，Newsgator properties(包括 Mac NetNewsWire)是 TechCrunch 用户第二常用的提要阅读器[，仅次于 FireFox。我认为该公司的路线图大体上预示了 RSS 联合的未来。Bloglines](https://web.archive.org/web/20220627213856/http://www.crunchnotes.com/?p=229) 最近有哪些创新？将博客搜索整合到 Ask.com，并在阅读器中支持 flash。在 Reinacker 的文章中讨论的 Newsgator 的后续步骤将会让你大吃一惊。我认为这些是我们将从该领域的其他供应商那里看到的功能和问题。

下文讨论的重点包括:

*   推荐订阅源
*   利基默认订阅选项
*   社交网络/关于订阅源的评论
*   RSS 无处不在——它还能去哪里？
*   通过电话订阅源和播客
*   广告，企业和私人标签的可能性。

 以下是我对路线图要点的总结和思考。

*   Reinacker 说，免费在线版的 [Newsgator](https://web.archive.org/web/20220627213856/http://newsgator.com/) 有很多功能，包括“更具互动性的 feed 发现机制(基于更大的用户群体及其订阅)。”Bloglines 充分利用用户订阅来消除博客搜索垃圾——只搜索 Bloglines 用户订阅的订阅源，排除垃圾博客订阅源。Reinacker 在这里一定是在暗示一些更主动的东西，可能是通过将你的订阅与其他读者的重叠订阅进行比较而确定的推荐源。换句话说，“你订阅了 [WorldChanging](https://web.archive.org/web/20220627213856/http://www.worldchanging.com/) ，很多其他订阅这个的读者也订阅了 [GlobalVoicesOnline](https://web.archive.org/web/20220627213856/http://www.globalvoicesonline.org/) 这种类型的东西已经是社会化书签领域的最佳实践了——在 RSS 中提供同样的东西是合乎逻辑的。ShareYourOPML 在极客聚居区就是这样做的。
*   “完全不同的用户界面范例(用户可以从不同的选项中选择，每个选项迎合不同类型的用户)”
    这可能意味着将自己描述为某种类型的用户，并根据您的兴趣向您提供默认的订阅选项(OPML 文件)。人们能够自动获取来自多个大众媒体垂直领域的高质量信息，这将是一件好事。我希望看到国际新闻焦点、环境焦点、科学焦点——为什么不呢？Newsgator 已经拥有所有主要在线阅读器供应商中最好的 OPML 支持，他们应该利用这一点。
*   弱化术语 RSS 提要。Reinacker 说大多数人不想看到这个缩写。我确信这是真的，我也很想让尽可能多的人使用 RSS，但是这样说总是让我担心功能会减少。我希望这不是系统前进的方向。
*   Rieneker 写道，改变人们发现提要的方式，在发现点增加价值，并参与更大的用户生态系统。这是社交网络的话题，鉴于 Newsgator 庞大的企业用户群可以自我反馈，没有理由不能对其进行评论、推荐等。期待 Newsgator 更多地以对象本身的形式呈现提要；包含动态更新信息但也由静态 URL 标识的对象，这些对象可以通过多种方式进行注释和关联。
*   最近发布的用于雅虎的 Newsgator 插件。Messenger 被强调为“无处不在的新闻组”的一个例子。在线 RSS 阅读器领域的每一个主要供应商都已经有了移动版本——除了浏览器、桌面、移动设备和现在的 IM 之外，feed 阅读器还能去哪里呢？当 Reinacker 说到处都有更多的 Newsgator 出现时，他是什么意思？我想不出任何地方。
*   用于 Windows Mobile 手机的 Newsgator Mobile 最近推出，但 Reinacker 说，所有支持 java 的手机将很快能够使用 Newsgator Mobile。让我们希望它不会像 Newsgator Online 几个月来那样被大量的信息堵塞。Reinacker 还表示，该公司将很快提供移动音频和视频播客。这将是对通过电话收听播客范式的一次重要考验——预计未来将会很大。
*   Reinacker 表示，该公司已经对 Newsgator API 和改进的分析做出了重大承诺。我将在这里稍微提到这一点，但就第三方服务而言，其影响可能很大。这也可能意味着更多的 RSS 广告，这对用户来说是好是坏，取决于它是如何实现的。
*   在 Newsgator enterprise 服务上，Reinacker 说“这里有太多的活动，我甚至不确定我是否能做得好。”Newsgator enterprise 服务是该公司的一大收入来源。由于 TechCrunch 不是一个专注于企业的博客，我将把感兴趣的读者引向 Reinacker 文章的倒数第二段。他提到了改进的门户整合、群组剪辑(企业标签？)和非常小的商业工具。首先在日本上市，今年晚些时候在美国和欧洲上市。
*   Newsgator 私人标签是 Newsgator 服务中最酷的部分之一。Newsgator [MyUSAToday](https://web.archive.org/web/20220627213856/http://my.usatoday.com/news/myusatoday.aspx) 和 Newsgator [MyNewsweek](https://web.archive.org/web/20220627213856/http://headlines.newsweek.com/NGWhiteLabel/Sites/NWk/reader.aspx?feedid=174011&cnt=3) 本身可能并不令人兴奋，但想想它们的潜力:一个托管的社区订阅源阅读器，种子是由主题专家管理员选择的订阅源。在社区主题上添加一个独特的标签为编辑收藏夹的条目，你就有了一个强大的超级动态的在线门户。但是请坚持，不久前我打电话给 Newsgator 销售热线询问这个产品的报价(没有透露任何组织名称)，但是没有收到他们的回复。这对 Newsgator 来说是一大笔钱，但不幸的是，他们的路线图只关注未来的广告和公关可能性——尽管公关[可能是一件好事](https://web.archive.org/web/20220627213856/http://forimmediaterelease.biz/)。

RSS 是几乎所有 Web 2.0 的基础，不是吗？这使得博客的读者群可以扩展，播客可以订阅，维基的变化可以观看等等。如果 Newsgatgor 能够成功地提供这一路线图所提到的创新功能，而不陷入粗俗的商业主义陷阱，Reinacker 的愿景可能会对媒体的未来产生深远的影响。
