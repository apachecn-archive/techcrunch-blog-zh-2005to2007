# TalkCrunch:亚马逊 S3 网格存储团队播客

> 原文：<https://web.archive.org/web/http://www.techcrunch.com:80/2006/03/26/talkcrunch-podcast-with-amazon-s3-grid-storage-team/>

[](https://web.archive.org/web/20161107045607/http://www.talkcrunch.com/)[talk crunch 第三集](https://web.archive.org/web/20161107045607/http://www.talkcrunch.com/2006/03/24/episode-3-amazons-new-grid-storage-service/)到了。我们邀请了亚马逊 S3 团队的两位关键成员，Adam Selipsky(亚马逊网络服务产品管理和开发者关系副总裁)和 Dave Barth(S3 产品经理)，来谈论亚马逊令人兴奋的新网格存储网络服务。

[正如我之前所写的](https://web.archive.org/web/20161107045607/http://www.techcrunch.com/2006/03/14/amazon-grid-storage-web-service-launches/)，S3 为那些对存储用户界面有很好想法的初创公司提供了一个绝佳的机会，以避免构建后端存储基础设施。亚马逊提供极低的价格和非常可靠的基础设施。对于一些人来说，S3 将允许他们推出一项他们原本无法打造的服务。

初创公司 Omnidrive 的首席执行官尼克·库布里洛维奇(Nik Cubrilovic)是该节目的共同主持人(基思·蒂尔也是)，我们试图说服他转到 S3 做后端存储，但没有成功。

Talkcrunch [在前三场演出后继续表现良好](https://web.archive.org/web/20161107045607/http://www.crunchnotes.com/?p=176)。它目前被列入 iTunes“新的和值得注意的”部分，并且是前 20 名技术播客之一(目前排名第 12)。
