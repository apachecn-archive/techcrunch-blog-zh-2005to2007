谷歌嘲讽Skype，发布GTalk API

原文：https://web.archive.org/web/http://www.techcrunch.com/2005/12/17/google-talk-puts-gun-to-skypes-head/trackback/

谷歌刚刚发布了一套名为 Libjingle 的组件，允许第三方应用程序与 Google Talk 互动。这些组件，包括一些源代码，在一个非常自由的许可下发布，允许自由整合到商业和非商业软件中。

我们发布这个源代码是我们致力于在基于互联网的实时通信中促进消费者选择和互操作性的一部分。谷歌的源代码是在伯克利式的许可下提供的，这意味着你可以自由地将它合并到商业和非商业软件中并分发它。

除了支持与Google Talk的互操作性，库中还有几个通用组件，如P2P堆栈，可用于构建各种通信和协作应用程序。我们渴望看到社区将利用这项技术构建的许多创新应用。

下面是库的各个组件的摘要。您可以使用这些组件中的任何一个或全部。

*基础–低级便携式实用功能。
* p2p——P2P堆栈，包括基本的P2P功能和XMPP的客户端挂钩。
*会话–电话呼叫信令。
*第三方–某些功能需要非Google组件。
* xmllite–XML解析器。
* XMPP–XMPP发动机。

谷歌在推动谷歌地图混搭方面做得很好——一些人声称有成千上万的第三方应用。这次代码发布更加重要，是Skype的一个巨大的终结。事实上，谷歌已经用枪指着Skype的脑袋，要求他们现在也发布他们的API。

Skype允许与Skype客户端集成。谷歌所做的远远不止这些，它允许与Google Talk的VOIP网络整合。

这里有一个潜在混搭的例子——将谷歌地图位置信息放在与google talk相同的应用程序中，允许对对话中的每个人进行映射( Gizmo 就是这样做的)。看看当地的企业和人们，把他们加入到对话中。
