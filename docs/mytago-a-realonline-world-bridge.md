# Mytago:真实的/在线的世界桥梁

 [](https://web.archive.org/web/20220929220420/http://www.mytago.com/) 和蚊子一样， [Mytago](https://web.archive.org/web/20220929220420/http://www.mytago.com/) 正试图为人们寻找一种简单的方法来标记真实世界的东西，以便在线互动。

这个想法是创建一个视觉标签(见图片)，可以包含在网站上，或者更有用的是，像海报一样离线。查看 Mytago 标签的人可以用他们的照相手机给它拍照，发邮件或者上传到 Mytago。与 Mytago 标签相关联的任何元数据都将在用户的帐户上添加书签。用户可以通过电子邮件发送标签中包含的代码，而不是拍摄图片。

每个标签都有统计数据，因此创建者可以看到有多少人给它加了书签。用户可以对任何标签发表评论。

米塔戈是赢家吗？我不这么认为。鉴于建立一个网站是如此容易，我不明白为什么人们不干脆建立一个网站，提供他们喜欢的任何信息，并在他们的离线海报或其他东西中包含一个网址。

我写这个是因为我喜欢实验，Mytago 引起了我的注意。我认为 [Mozes](https://web.archive.org/web/20220929220420/http://www.mozes.com/) 解决方案，以一种不同的方式解决这个市场，并且不需要任何视觉标签，可能会有优势。我对 [Mozes 的评论在这里](https://web.archive.org/web/20220929220420/http://www.beta.techcrunch.com/2006/04/20/mozes-secure-your-keyword/)。

**更新:** Nana 和 Mark Devlin 发表评论，谈论日本使用的[二维码](https://web.archive.org/web/20220929220420/http://en.wikipedia.org/wiki/QR_Code)，这种二维码看起来非常类似于 Mytago 标签，可以通过手机读取。让你的手机直接理解代码的能力让这一切变得更加有趣。我们将会看到美国是否会出现类似的情况。
